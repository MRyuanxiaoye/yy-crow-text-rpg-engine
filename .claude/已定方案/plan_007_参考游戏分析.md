# 参考游戏分析

棋盘系统设计的参考游戏调研。

## 1. Suzerain（最接近参考）

- 类型：文字政治RPG，扮演虚构国家总统
- 核心交互：对话为主体（和部长/顾问/反对派对话做决策）
- 棋盘设计：不是一张统一面板，而是多层信息源并行
  - 对话是主体
  - 报纸头条持续滚动（世界在你决策之外也在变化）
  - 政治地图展示区域状态
  - 预算/经济数据作为约束
  - 人物档案库随时可查背景资料
- 关键设计：信息不一次给完，必须通过对话和事件逐步拼凑全貌
- 与我们的关联：验证了'文字对话+结构化状态面板'的组合可行性，报纸机制=我们的被动事件/NPC消息推送

## 2. King of Dragon Pass / Six Ages（叙事策略鼻祖）

- 类型：叙事策略游戏，管理一个部落
- 核心交互：事件/场景卡片式选择（近600个交互场景）
- 棋盘设计：
  - 部落资源面板常驻可见（人口/牛群/粮食/财货/战士）
  - 顾问团有独立性格，对同一事件给出不同建议
  - 每年5个季节，每季可做2个宏观决策
  - 周边区域地图展示邻居关系
- 关键设计：文本量巨大（相当于前四本哈利波特），但通过场景卡片+常驻面板实现'叙事+策略'平衡
- 与我们的关联：顾问团=我们的NPC系统，季节制=我们的回合制，资源面板=我们的维度面板。验证了纯文字+面板的策略深度

## 3. Reigns（极简棋盘极致）

- 类型：卡片式王国管理
- 核心交互：左滑右滑做二选一决策
- 棋盘设计：
  - 四个资源条永远可见（军队/人民/教会/财富）
  - 滑动时可预览每个选项对四个维度的影响方向和幅度
  - 任何维度满或空=死亡
- 关键设计：决策前预览后果，极简但信息完整
- 与我们的关联：预览影响=我们的'对话内嵌数值反馈'。四维度永远可见=我们的维度面板。验证了最少信息也能支撑策略决策

## 4. Pandemic Legacy Season 2（地图迷雾桌游范例）

- 类型：合作桌游，末日后世界
- 核心交互：桌游回合制行动
- 棋盘设计：
  - 开局地图大部分区域是空白的（只有太平洋沿岸几个城市）
  - 通过探索行动逐步揭开新区域（拆密封包、贴地图贴纸）
  - 信息本身就是奖励和进展的体现
- 关键设计：空白区域不是没有东西，是'你知道那里有东西但看不到'
- 与我们的关联：直接验证了迷雾机制——好奇心来自已知和未知的边界。情报维度=探索成本，揭开迷雾=情报投资回报

## 5. Crusader Kings III / Old World（叙事+策略+地图结合）

- 类型：大战略游戏
- CK3：封建领主管理，地图是政治关系的空间化展示
- Old World：文明4设计师作品，CK3式角色事件+文明式地图
- 棋盘设计：
  - 地图是基础层（领土/军队/资源）
  - 角色事件以文字卡片弹出在地图之上
  - 角色关系/性格/密谋构成第二层信息
- 关键设计：策略层（地图）和叙事层（事件/对话）是两个独立但交互的系统
- 与我们的关联：验证了'结构化世界状态+自由文字事件'的双层架构

## 核心结论

| 参考游戏 | 我们学什么 |
|---------|----------|
| Suzerain | 多信息源并行，信息渐进式揭示 |
| KoDP/Six Ages | 常驻资源面板+场景卡片+顾问建议=完整策略体验 |
| Reigns | 决策前预览后果，最少信息支撑策略 |
| Pandemic Legacy S2 | 迷雾机制，好奇心=已知与未知的边界 |
| CK3/Old World | 结构化状态+自由叙事的双层架构 |
