# 阶段2：权重引擎（Weight Engine）

## 在方法链中的位置

```
  阶段0：数据预处理与特征工程
        ↓
  阶段1：排除引擎
        ↓
【阶段2：权重引擎】 ← 你在这里
        ↓
  阶段3：组合生成器
        ↓
  阶段4：购买优化器
```

## 为什么是第三步（紧接在排除引擎之后）？

### 起因
经过阶段1的排除，候选号码池已经从全量（如35个）缩小到了约50%-70%（如18-25个）。但这些保留下来的号码并不是"同等可能出现"的——有些号码各方面指标都很好（频率回归、遗漏值合理、转移概率高），有些只是"没有被排除"而已。我们需要一种方法来**区分这些号码的优先级**。

### 目的
对每个未被排除的号码计算一个**综合权重分数**（0到1之间），分数越高表示该号码在下一期出现的可能性越大。这个权重分数将直接驱动阶段3的组合生成——高权重号码更容易被选入候选组合。

### 为什么必须在排除之后？
1. **候选池更小**：排除后只剩18-25个号码，权重计算的精度更高（噪声更少）
2. **避免浪费**：不需要为已排除的号码计算权重
3. **信号更清晰**：排除掉"明显不可能"的号码后，剩余号码之间的权重差异更有意义

### 为什么不能跳过直接生成组合？
如果没有权重，组合生成器只能在候选号码中**均匀随机**选择，这等于放弃了所有统计信息。权重引擎的价值在于：把"哪些号码更可能出现"这个判断量化为数字，让组合生成器可以**有偏向地**选择号码。

---

## 核心设计：多算法归一化融合

与排除引擎类似，权重引擎也是多算法并行计算后融合。但融合方式不同：

- 排除引擎：加权投票 → 二元决策（排除/保留）
- 权重引擎：归一化加权求和 → 连续分数（0~1）

```
                    ┌── 算法2.1：频率回归权重 ────┐
                    ├── 算法2.2：遗漏值回归权重 ──┤
特征矩阵 ──────────→├── 算法2.3：时序衰减权重 ────┼──→ 归一化加权融合 ──→ 号码权重表
+ 未排除号码        ├── 算法2.4：马尔可夫转移权重 ─┤
                    ├── 算法2.5：共现关联权重 ────┤
                    └── 算法2.6：深度学习预测权重 ─┘
```

---

## 算法2.1：频率回归权重

### 设计起因
大数定律告诉我们：随着样本量增大，每个号码的出现频率会趋近于理论概率。如果一个号码的历史出现频率**长期低于**理论概率，那么从统计回归的角度看，它未来出现的概率倾向于升高（以"补偿"之前的低频）。反之，长期高于理论概率的号码，未来可能会"降温"。

**注意**：这不是赌徒谬误。赌徒谬误是说"连续出红所以下次一定出黑"，那是对独立事件的错误推理。频率回归权重是基于**长期统计偏差**的修正，是一种合理的贝叶斯先验更新。

### 目的
识别那些"历史频率低于理论值"的号码，给予更高的权重。

### 逻辑

```
输入：
  freq[v] = 号码 v 的历史出现频率（出现次数 / 总期数）
  theoretical_prob = 选取个数 / 号码总数
    大乐透前区：5/35 ≈ 0.143
    大乐透后区：2/12 ≈ 0.167
    双色球红球：6/33 ≈ 0.182
    双色球蓝球：1/16 = 0.0625

处理：
  步骤1：计算频率偏差
    deviation[v] = theoretical_prob - freq[v]
    正值 → 频率低于理论值（"欠"了）
    负值 → 频率高于理论值（"多"了）

  步骤2：映射为权重
    raw_weight[v] = sigmoid(deviation[v] * scale_factor)
    其中 scale_factor 控制灵敏度（建议值：10~20）

    sigmoid 函数将偏差映射到 (0, 1) 区间：
      偏差为0 → 权重 = 0.5（中性）
      偏差为正（频率偏低）→ 权重 > 0.5
      偏差为负（频率偏高）→ 权重 < 0.5

输出：
  每个号码的频率回归权重（0~1之间）
```

### 边界情况
- 数据量太少时（<50期），频率偏差不稳定，该算法的融合权重应降低
- 某些号码可能因为规则变更（如号码范围调整）导致频率异常，需要只使用规则变更后的数据

## 算法2.2：遗漏值回归权重

### 设计起因
每个号码都有一个"自然出现节奏"——平均每隔几期出现一次。当一个号码的遗漏值（连续未出现期数）接近或超过其平均间隔时，它"该出现了"的概率在统计上是升高的。这可以用几何分布的累积概率来建模。

### 目的
根据号码的当前遗漏值和历史平均间隔，计算它"即将出现"的概率作为权重。

### 逻辑

```
输入：
  missing[v] = 号码 v 的当前遗漏值
  avg_gap[v] = 号码 v 的历史平均间隔期
  max_gap[v] = 号码 v 的历史最大间隔期

处理：
  步骤1：估计单期出现概率
    p[v] = 1 / avg_gap[v]

  步骤2：计算累积概率（几何分布）
    cumulative_prob = 1 - (1 - p[v]) ^ missing[v]
    含义：在 missing[v] 期内至少出现一次的概率

  步骤3：冷号修正
    如果 missing[v] > max_gap[v] * 1.2:
      该号码可能是"真冷号"（不是正常波动，而是系统性变冷）
      对权重做衰减：
        cold_factor = exp(-(missing[v] - max_gap[v]) / max_gap[v])
        raw_weight[v] = cumulative_prob * cold_factor
    否则:
        raw_weight[v] = cumulative_prob

输出：
  每个号码的遗漏值回归权重（0~1之间）
```

### 与排除引擎的关系
- 排除引擎的算法1.2排除"遗漏值过小"的号码（刚出现不久）
- 权重引擎的算法2.2给"遗漏值适中偏大"的号码加权
- 两者形成互补：排除引擎去掉"太新鲜"的，权重引擎加分"该出现"的
- 被排除的号码不会进入权重引擎，所以不存在冲突

### 冷号修正的意义
不加修正的话，遗漏值越大权重越高，但现实中有些号码可能因为某种原因（如号码池调整后的冷门号）长期不出现。冷号修正避免了对这类号码的过度加权。

## 算法2.3：时序衰减权重

### 设计起因
直觉上，最近几期的数据比很久以前的数据更有参考价值。一个号码如果在最近10期内出现了3次，和它在100期前的10期内出现了3次，前者的信号强度应该更高。时序衰减权重通过指数衰减函数，让近期数据的影响力大于远期数据。

### 目的
捕捉号码的"近期热度"，给最近频繁出现的号码更高的权重。

### 逻辑

```
输入：
  对于号码 v，最近 W 期的出现记录：
    appearances[v] = [a₁, a₂, ..., aW]
    其中 aᵢ = 1（第i期出现）或 0（未出现）
    i=1 是最近一期，i=W 是最远一期

  衰减系数 λ（可调参数，建议值：0.05~0.15）
  窗口大小 W（可调参数，建议值：30~50期）

处理：
  步骤1：计算衰减权重序列
    decay_weights = [exp(-λ * i) for i in range(W)]
    含义：第1期（最近）权重最大，越远权重越小

  步骤2：加权求和
    raw_weight[v] = Σ(appearances[v][i] * decay_weights[i]) for i in range(W)

  步骤3：归一化
    max_possible = Σ(decay_weights)  # 如果每期都出现的理论最大值
    normalized_weight[v] = raw_weight[v] / max_possible

输出：
  每个号码的时序衰减权重（0~1之间）
```

### 与排除引擎的制衡关系
- 排除引擎的算法1.1会排除"连续出现多期"的号码
- 权重引擎的算法2.3会给"近期频繁出现"的号码加权
- 这看似矛盾，实际上是精妙的制衡：
  - 连续出现3期以上的号码 → 被排除引擎排除，不进入权重引擎
  - 近期出现过1-2次但没有连续的号码 → 通过排除引擎，在权重引擎中获得加分
  - 这样就区分了"过热"（排除）和"适度热"（加权）

### 参数敏感性
- λ 越大，衰减越快，越重视最近几期（短期视角）
- λ 越小，衰减越慢，越重视长期趋势（长期视角）
- W 越大，考虑的历史越长，但计算量也越大
- 建议通过回测找到最优的 λ 和 W 组合

## 算法2.4：马尔可夫转移权重

### 设计起因
马尔可夫转移矩阵记录了"上一期出现号码A后，下一期出现号码B"的历史概率。排除引擎已经用它排除了转移概率极低的号码，但转移矩阵中还有大量有价值的信息没有被利用——那些转移概率**较高**的号码，应该获得更高的权重。

### 目的
从"号码接力"的角度，给那些从上一期号码出发转移概率较高的号码加权。

### 逻辑

```
输入：
  transition_matrix = 阶段0计算的转移矩阵
    transition_matrix[i][j] = P(下一期出j | 上一期出i)
  last_draw = 上一期开奖号码（如 [5, 12, 18, 23, 35]）
  second_last_draw = 上上期开奖号码（用于二阶扩展）

处理：
  对于每个候选号码 v（未被排除的）：

    一阶转移权重：
      probs_1 = [transition_matrix[i][v] for i in last_draw]
      weight_order1 = mean(probs_1)

    二阶转移权重（可选扩展）：
      对于上上期的每个号码 j 和上一期的每个号码 i：
        计算二阶条件概率（需要更大的数据量支撑）
      weight_order2 = 二阶条件概率的均值

    最终权重：
      raw_weight[v] = 0.7 * weight_order1 + 0.3 * weight_order2
      （如果数据量不足以支撑二阶，则 raw_weight[v] = weight_order1）

输出：
  每个号码的马尔可夫转移权重（0~1之间，需后续归一化）
```

### 与排除引擎算法1.4的关系
- 共享同一个转移矩阵，但使用方向相反
- 排除引擎：转移概率 < 阈值 → 排除
- 权重引擎：转移概率越高 → 权重越高
- 被排除的号码不会进入权重引擎，所以这里只处理"通过排除"的号码

## 算法2.5：共现关联权重

### 设计起因
历史数据中，某些号码经常在同一期出现（"结伴"现象）。比如号码8和号码23可能在历史上共同出现了15次，远高于随机期望。如果当前权重引擎已经给号码8打了高分，那与8经常共现的号码23也应该获得加分。这类似于推荐系统中的"协同过滤"思想。

### 目的
利用号码之间的共现关系，对高权重号码的"伙伴号码"进行加权。

### 逻辑

```
输入：
  co_occurrence_matrix = 阶段0计算的号码共现矩阵
    co_occurrence_matrix[i][j] = 号码i和j在同一期出现的次数
  current_weights = 其他算法（2.1~2.4）已经计算出的初步权重

处理：
  步骤1：选取锚点号码
    top_k_numbers = 当前权重排名前 K 的号码（K=5~8）
    这些是"大概率会出现"的号码，作为共现分析的锚点

  步骤2：计算共现加分
    对于每个候选号码 v：
      co_score = 0
      for u in top_k_numbers:
        co_score += co_occurrence_matrix[v][u] * current_weights[u]
      co_score /= sum(current_weights[u] for u in top_k_numbers)

    含义：v 与高权重号码的共现越频繁，加分越多
          且共现的"伙伴"权重越高，加分越多

  步骤3：迭代收敛（可选）
    将 co_score 加入 current_weights，重新选取 top_k
    重复步骤1-2，运行2-3轮直到权重变化小于阈值

输出：
  每个号码的共现关联权重（0~1之间，需后续归一化）
```

### 独特价值
- 这是唯一一个考虑**号码间关系**的权重算法
- 其他算法（2.1~2.4）都是独立评估每个号码，忽略了号码之间的关联
- 共现关联权重弥补了这个缺陷，引入了"组合视角"
- 迭代机制让高权重号码之间形成"互相增强"的效果

### 风险与控制
- 共现关系可能是偶然的（样本量不足时），需要设置最低共现次数阈值
- 迭代次数不宜过多（2-3轮），否则可能导致权重过度集中在少数号码上
- 融合权重设为0.15，不宜过高，避免共现关系主导最终结果

## 算法2.6：深度学习预测权重（必选，Apple M3 CPU）

### 设计起因
前5个算法都是基于**人工设计的统计规则**——频率回归、遗漏值分布、指数衰减等。这些规则虽然有统计学依据，但它们只能捕捉人类能想到的模式。深度学习模型（LSTM/Transformer）的优势在于：它可以从数据中**自动学习**人类未必能发现的复杂非线性模式。

### 目的
使用深度学习模型，输入历史序列特征，直接输出每个号码在下一期出现的概率分布。作为管道中**不可缺少的核心算法**，与其他5个统计算法形成互补。

### 为什么是必选而非可选
- 统计算法只能捕捉线性或简单非线性关系
- 深度学习可以捕捉高阶交互特征和长程依赖
- 在回测中，深度学习算法通常能提供与统计算法**不相关的信号**，融合后显著提升整体表现
- Apple M3 芯片的 MPS 后端足以支撑本项目规模的模型训练和推理

### 硬件方案：Apple M3

```
硬件：Apple M3 芯片
加速后端：PyTorch MPS (Metal Performance Shaders)

M3 的优势：
  - 统一内存架构：CPU和GPU共享内存，无需数据拷贝
  - Metal GPU 加速：通过 MPS 后端，PyTorch 可利用 M3 的 GPU 核心
  - 能效比高：适合长时间训练任务

适配策略：
  - 模型规模控制在中小型（参数量 < 5M）
  - batch_size 适当缩小（16~32）
  - 使用 float32 精度（M3 的 float16 支持有限）
  - 训练数据量不大（几千期），M3 完全可以胜任

PyTorch 设备选择：
  device = torch.device("mps" if torch.backends.mps.is_available() else "cpu")
```

### 模型架构

```
方案A：Transformer Encoder（推荐）

  输入层：
    最近 N 期的特征矩阵 (batch, N, feature_dim)
    N = 序列长度（建议30~50期）
    feature_dim = 每期的特征维度（号码特征 + 组合特征，约20~40维）

  位置编码：
    可学习的位置嵌入（Learnable Positional Embedding）
    让模型知道每期数据的时间顺序

  Transformer Encoder：
    层数：4层
    注意力头数：4头（M3上4头比8头更高效）
    隐藏维度：128
    前馈维度：256
    Dropout：0.2

  输出层：
    全连接层 → 每个号码的出现概率
    大乐透前区：输出维度 35（每个号码一个概率值）
    大乐透后区：输出维度 12
    使用 Sigmoid 激活（多标签分类，每个号码独立判断）

方案B：LSTM（备选，更简单）

  输入层：同上
  LSTM：2层，hidden_size=128，bidirectional=False
  输出层：同上

选择建议：
  优先使用 Transformer，因为：
  - 自注意力机制天然适合捕捉号码间的位置关系
  - 并行计算效率更高（M3 上更快）
  - 注意力权重可用于可解释性分析
```

### 训练策略

```
数据准备：
  使用滑动窗口生成训练样本
  窗口大小 = N 期
  标签 = 第 N+1 期的开奖号码（多热编码）

  示例（大乐透前区）：
    输入：第1~30期的特征矩阵
    标签：第31期的号码 [5,12,18,23,35] → [0,0,0,0,1,0,...,1,...,1,...,1,...,1]
                                            （35维的多热向量）

损失函数：
  多标签二元交叉熵（BCE Loss）
  loss = -Σ(yᵢ * log(pᵢ) + (1-yᵢ) * log(1-pᵢ))

  可选附加项：
    + α * 唯一性惩罚（鼓励输出概率分布更集中）
    + β * 数量约束（鼓励高概率号码数量接近实际选取数）

训练参数：
  优化器：AdamW (lr=1e-3, weight_decay=1e-4)
  学习率调度：CosineAnnealingLR
  Epochs：100~200
  Batch size：16~32（M3 内存友好）
  早停：验证集损失连续10个epoch不下降则停止

验证策略：
  时间序列交叉验证（不能随机划分，必须按时间顺序）
  训练集：前80%的期数
  验证集：后20%的期数
```

### 推理流程

```
输入：最近 N 期的特征矩阵
输出：每个号码的出现概率

  probs = model(recent_N_features)  # shape: (num_range,)

  对于大乐透前区：
    probs = [p₁, p₂, ..., p₃₅]  # 每个号码的出现概率

  将概率作为权重：
    raw_weight[v] = probs[v]

注意：
  模型输出的概率已经是归一化的（Sigmoid），可直接作为权重
  但在融合时仍需与其他算法的权重做统一归一化
```

### 可解释性
- Transformer 的注意力权重可以可视化，展示模型"关注"了哪些历史期数
- 可以分析模型对不同特征的敏感度（通过特征重要性分析）
- 这些信息有助于理解模型的决策逻辑，也有助于发现新的统计规律

## 融合器：归一化加权求和

### 设计起因
6个权重算法各自输出的分数量纲不同——频率回归输出的是sigmoid值（0~1），遗漏值回归输出的是累积概率（0~1），深度学习输出的是模型概率（0~1），但它们的分布形态差异很大。直接加权求和会导致某些算法因为数值范围大而主导结果。需要先归一化，再融合。

### 目的
将6个算法的输出统一到同一量纲，然后加权融合为每个号码的最终权重分数。

### 逻辑

```
输入：
  对于每个候选号码 v，6个算法各自输出一个原始分数：
    raw_scores[v] = {
      "freq_regression":    s₁,
      "missing_regression": s₂,
      "time_decay":         s₃,
      "markov":             s₄,
      "co_occurrence":      s₅,
      "deep_learning":      s₆,
    }

  算法融合权重（通过回测调优）：
    algo_weights = {
      "freq_regression":    0.20,
      "missing_regression": 0.20,
      "time_decay":         0.15,
      "markov":             0.15,
      "co_occurrence":      0.15,
      "deep_learning":      0.15,
    }

处理：
  步骤1：每个算法内部做 Min-Max 归一化
    对于算法 a：
      min_a = min(raw_scores[v][a] for v in all_candidates)
      max_a = max(raw_scores[v][a] for v in all_candidates)
      normalized[v][a] = (raw_scores[v][a] - min_a) / (max_a - min_a)

    含义：每个算法的分数都映射到 [0, 1]，最高分的号码为1，最低分为0

  步骤2：加权求和
    for v in all_candidates:
      combined[v] = Σ(algo_weights[a] * normalized[v][a]) for a in algorithms

  步骤3：全局 Softmax 归一化
    temperature = T（可调参数，建议值：0.5~2.0）
    final_weight[v] = exp(combined[v] / T) / Σ(exp(combined[u] / T))

    温度参数的作用：
      T < 1 → 分布更尖锐，高权重号码更突出
      T = 1 → 标准softmax
      T > 1 → 分布更平坦，号码间差异缩小

输出：
  number_weights = {v: final_weight[v] for v in all_candidates}
  所有权重之和为1，可直接作为概率分布使用
```

### 权重调优
- 6个算法的融合权重通过回测自动调优
- 目标函数：最大化 `top10_hit_rate`（权重Top10号码中实际出现的个数）
- 约束：所有权重之和为1，每个权重在 [0.05, 0.35] 范围内
- 温度参数 T 也纳入调优范围

### 核心指标
- **权重相关性**：号码权重与实际出现的点二列相关系数（目标 > 0.3）
- **Top5命中率**：权重Top5号码中实际出现的平均个数（目标 > 1.0）
- **Top10命中率**：权重Top10号码中实际出现的平均个数（目标 > 2.0）
- **平均排名**：实际出现号码的平均权重排名（越小越好）

## 与上下游的衔接

### 从阶段1接收
- `remaining_numbers`：未被排除的号码列表（只对这些号码计算权重）
- 阶段0的全部特征数据（通过管道传递）

### 向阶段3传递
- `number_weights`：每个候选号码的最终权重（归一化后，和为1）
- `weight_details`：每个号码的权重明细（6个算法各自的贡献）
- `top_numbers`：按权重降序排列的号码列表

阶段3的组合生成器将直接使用 `number_weights` 作为采样概率分布，高权重号码更容易被选入候选组合。
