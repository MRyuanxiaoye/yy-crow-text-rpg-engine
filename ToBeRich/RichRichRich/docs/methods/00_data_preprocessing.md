# 阶段0：数据预处理与特征工程

## 在方法链中的位置

```
【阶段0：数据预处理与特征工程】 ← 你在这里
        ↓
  阶段1：排除引擎
        ↓
  阶段2：权重引擎
        ↓
  阶段3：组合生成器
        ↓
  阶段4：购买优化器
```

## 为什么是第一步？

### 起因
原始开奖数据只是一组组号码记录（如 `[5, 12, 18, 23, 35]`），这种原始形态对后续的排除引擎和权重引擎来说是**不可直接使用的**。排除引擎需要知道"这个号码最近出现了几次连续"、"这个号码的遗漏值是多少"；权重引擎需要知道"这个号码的历史频率偏差"、"与其他号码的共现关系"。这些信息都不存在于原始数据中，必须通过计算提取出来。

### 目的
将原始开奖记录转化为一份**统一的特征矩阵**，供后续所有阶段共享使用。这样做有三个好处：
1. **避免重复计算**：排除引擎和权重引擎都需要频率、遗漏值等数据，统一计算一次即可
2. **保证数据一致性**：所有阶段看到的是同一份特征数据，不会出现各算法各自计算导致的不一致
3. **支持增量更新**：新一期数据到来时，只需更新特征矩阵，不需要每个算法各自重新处理原始数据

### 为什么不能跳过这一步？
如果没有阶段0，每个排除算法和权重算法都需要自己从原始数据中提取特征，这会导致：
- 大量重复计算（6个排除算法 + 6个权重算法 = 12次重复提取）
- 各算法对同一特征的计算方式可能不一致
- 新增算法时需要重新编写数据处理逻辑

```

  transition_matrix:  马尔可夫转移矩阵,
}
```

## 四大特征提取模块

### 模块0.1：基础特征提取

**提取什么**：每个号码的"个人档案"

```
对于每个号码 v（如大乐透前区的 1~35）：

  历史出现频率 = v 出现的总次数 / 总期数
    含义：这个号码在历史上平均多久出现一次

  当前遗漏值 = 从最近一期往回数，v 已经连续多少期没出现
    含义：这个号码"沉默"了多久

  平均间隔期 = v 所有相邻两次出现之间的期数的平均值
    含义：这个号码正常情况下多久出现一次

  最大间隔期 = v 历史上最长的连续未出现期数
    含义：这个号码最多能"沉默"多久

  连续出现次数 = 从最近一期往回数，v 连续出现了几期
    含义：这个号码是否处于"热"状态
```

**为什么需要这些特征**：
- `遗漏值` → 排除引擎的"遗漏值异常排除器"和权重引擎的"遗漏值回归权重"都依赖它
- `连续出现次数` → 排除引擎的"连续重复排除器"依赖它
- `历史频率` → 权重引擎的"频率回归权重"依赖它
- `平均间隔期` → 排除引擎和权重引擎的多个算法都用它作为基准线

### 模块0.2：组合特征提取

**提取什么**：每一期开奖号码作为一个整体的特征

```
对于每一期的号码组合（如 [5, 12, 18, 23, 35]）：

  奇偶比 = 奇数个数 : 偶数个数（如 3:2）
    含义：这组号码的奇偶分布是否均衡

  大小比 = 大号个数 : 小号个数（以中位数为界）
    含义：这组号码偏大还是偏小

  区间分布 = 将号码范围分为3-4个区间，统计每个区间的号码个数
    含义：号码是否集中在某个区间

  和值 = 所有号码之和
    含义：这组号码的整体大小水平

  跨度 = 最大号码 - 最小号码
    含义：号码的分散程度

  连号组数 = 相邻号码的组数（如 [5,6] 算一组连号）
    含义：是否有号码"扎堆"

  AC值 = 所有号码两两之差的绝对值的去重个数 - (选号个数 - 1)
    含义：号码组合的复杂度，AC值越高越"散"
```

**为什么需要这些特征**：
- 排除引擎的"极端组合约束排除器"需要知道历史上奇偶比、大小比、和值的分布范围，才能判断什么是"极端"
- 组合生成器的"软约束检查器"需要这些统计数据来验证生成的组合是否合理

### 模块0.3：时序特征提取

**提取什么**：号码随时间变化的趋势

```
对于每个号码 v：

  滑动窗口统计：
    近5期出现次数、近10期出现次数、近20期出现次数、近50期出现次数
    含义：不同时间尺度下的"热度"

  出现趋势：
    对近 N 期的出现标记（0/1序列）做线性回归，取斜率
    斜率 > 0 → 趋势上升（越来越热）
    斜率 < 0 → 趋势下降（越来越冷）
    斜率 ≈ 0 → 平稳
    含义：这个号码是在变热还是变冷

  差分特征：
    相邻两期同一位置号码的差值
    含义：号码在该位置上的变化幅度

  周期性特征：
    对每个号码的出现序列做 FFT（快速傅里叶变换）
    提取主频率和对应的振幅
    含义：这个号码是否有周期性出现的规律
```

**为什么需要这些特征**：
- `滑动窗口统计` → 权重引擎的"时序衰减权重"依赖它
- `出现趋势` → 可作为权重引擎的辅助信号
- `周期性特征` → 排除引擎的"周期性排除器"依赖它

### 模块0.4：关联特征提取

**提取什么**：号码之间的关系

```
号码共现矩阵：
  co_occurrence[i][j] = 号码 i 和号码 j 在同一期出现的次数
  归一化后得到共现概率
  含义：哪些号码经常"结伴"出现

位置间相关性矩阵：
  对每个位置（如大乐透前区的5个位置），计算位置间的 Pearson 相关系数
  含义：某个位置出大号时，其他位置是否也倾向出大号

一阶马尔可夫转移矩阵：
  transition[i][j] = 上一期出现号码 i 后，下一期出现号码 j 的概率
  含义：号码之间的"接力"关系
```

**为什么需要这些特征**：
- `共现矩阵` → 权重引擎的"共现关联权重"依赖它
- `转移矩阵` → 排除引擎的"马尔可夫转移排除器"和权重引擎的"马尔可夫转移权重"共享它

## 实现要点

1. **缓存机制**：特征矩阵计算完成后缓存到本地文件，下次运行时如果数据没有更新则直接加载缓存
2. **增量更新**：新增一期数据时，只需更新受影响的特征（如遗漏值+1、频率重算），不需要全量重算
3. **前后区分离**：大乐透的前区（1-35）和后区（1-12）分别计算特征，因为它们是独立的号码池
4. **数据验证**：在特征提取前先验证原始数据的完整性和正确性（号码范围、个数、日期顺序等）

---

## 与下一阶段的衔接

阶段0的输出直接传入阶段1（排除引擎）和阶段2（权重引擎）：
- 排除引擎从 `number_features` 中读取遗漏值、连续出现次数等
- 排除引擎从 `transition_matrix` 中读取转移概率
- 排除引擎从 `combo_stats` 中读取组合特征的历史分布
- 权重引擎从 `number_features` 中读取频率、遗漏值、趋势等
- 权重引擎从 `correlation_matrix` 中读取共现关系

阶段0是整条管道的**数据基座**，它的质量直接决定后续所有阶段的表现。
