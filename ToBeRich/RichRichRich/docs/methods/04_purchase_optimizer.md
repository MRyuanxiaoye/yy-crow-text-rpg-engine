# 阶段4：购买优化器（Purchase Optimizer）

## 在方法链中的位置

```
  阶段0：数据预处理与特征工程
        ↓
  阶段1：排除引擎
        ↓
  阶段2：权重引擎
        ↓
  阶段3：组合生成器
        ↓
【阶段4：购买优化器】 ← 你在这里（终点）
```

## 为什么是最后一步？

### 起因
阶段3输出了50~200组高质量候选组合，但用户不可能全部购买——每注2元，200注就是400元。用户通常有一个预算（比如20元、50元），需要在预算内**精选**最有价值的组合。而且彩票有复式和胆拖玩法，合理利用这些玩法可以用更少的钱覆盖更多号码。

### 目的
在给定预算下，从候选组合中选出**最优购买方案**，实现"最少钱覆盖最多高权重号码"。这是整条管道的最终输出——一个可以直接去彩票站执行的购买方案。

### 为什么必须在组合生成之后？
- 购买优化是在**已有候选组合**中做选择，没有候选就无从优化
- 优化器需要知道每组候选的得分和号码构成，这些都来自阶段3
- 优化器还需要考虑组合之间的**覆盖关系**（两组号码重叠多少），这只有在组合确定后才能计算

### 为什么这一步不可缺少？
没有购买优化器，用户只能拿到一堆候选组合，然后自己决定买哪些。这有两个问题：
1. 用户无法快速判断哪些组合的"性价比"最高
2. 用户不了解复式/胆拖玩法的数学原理，无法利用这些玩法省钱

---

## 问题本质：集合覆盖问题

### 数学建模

```
定义：
  U = 所有高权重候选号码的集合
      （从阶段2的权重表中取权重Top-K的号码）
  S = {s₁, s₂, ..., sₙ} 为阶段3生成的候选组合集合
  每个 sᵢ ⊆ U，包含若干号码
  每个 sᵢ 有一个得分 score(sᵢ)
  每个 sᵢ 的购买成本 cost(sᵢ) = 2元

目标：
  选择 S 的一个子集 S*，使得：
    maximize  Σ(w(j) * covered(j))   # 覆盖号码的权重之和最大
    subject to  Σ(cost(sᵢ)) ≤ Budget  # 总花费不超过预算

这是一个 NP-hard 问题（加权集合覆盖问题的变体）
但候选组合数量有限（50~200），可以用近似算法高效求解
```

---

## 策略4.1：贪心集合覆盖

### 设计起因
贪心算法是集合覆盖问题最经典的近似算法，近似比为 ln(n)+1，在实践中表现很好。核心思想：每次选择"性价比最高"的组合——即每花1元钱能新覆盖最多权重的组合。

### 目的
快速找到一个不错的购买方案，作为基准方案。

### 逻辑

```
输入：
  candidates = 阶段3输出的候选组合列表（含号码和得分）
  number_weights = 号码权重表
  budget = 用户预算（元）

处理：
  covered = {}          # 已覆盖的号码集合
  selected = []         # 已选购买的组合
  budget_remaining = budget

  while budget_remaining >= 2 and candidates 还有未选的:

    步骤1：计算每个未选组合的"边际价值"
      对于每个未选组合 s：
        new_numbers = s.numbers - covered  # 新覆盖的号码
        new_weight = sum(number_weights[n] for n in new_numbers)
        marginal_value = new_weight * s.score
        含义：新覆盖号码的权重之和 × 组合自身得分

    步骤2：选择边际价值最高的组合
      best = argmax(marginal_value)

    步骤3：更新状态
      selected.append(best)
      covered = covered ∪ best.numbers
      budget_remaining -= 2

  return selected

输出：
  购买方案（组合列表 + 总花费 + 覆盖分析）
```

### 优缺点
- 优点：简单高效，O(n²)复杂度，结果有理论保证
- 缺点：贪心策略可能错过全局最优解
- 适用场景：作为基准方案，与其他策略对比

---

## 策略4.2：分层覆盖策略

### 设计起因
不是所有号码都同等重要——权重Top5的号码远比Top20的号码重要。贪心算法不区分号码的重要性层级，可能为了覆盖一个低权重号码而浪费预算。分层策略的思路是：**先确保最重要的号码被覆盖，再考虑次要的**。

### 目的
按号码重要性分层，优先保证高权重号码的覆盖率。

### 逻辑

```
输入：
  candidates = 候选组合列表
  number_weights = 号码权重表
  budget = 用户预算

处理：
  步骤1：将号码按权重分为三层
    第一层（核心）：权重排名前30%的号码
      这些号码必须被至少2组候选覆盖（双重保障）
    第二层（重要）：权重排名30%-60%的号码
      这些号码至少被1组候选覆盖
    第三层（补充）：权重排名60%-100%的号码
      有预算余量时才覆盖

  步骤2：覆盖第一层
    用贪心法选出能覆盖第一层所有号码的最少组合
    每个第一层号码要求被至少2组覆盖
    记录花费 cost_1

  步骤3：覆盖第二层
    在剩余预算 (budget - cost_1) 内
    用贪心法选出能覆盖第二层号码的组合
    记录花费 cost_2

  步骤4：覆盖第三层
    在剩余预算 (budget - cost_1 - cost_2) 内
    用贪心法选出能覆盖第三层号码的组合

  步骤5：合并所有选中的组合

输出：
  分层覆盖的购买方案
```

### 与策略4.1的区别
- 策略4.1对所有号码一视同仁
- 策略4.2优先保证高权重号码的覆盖
- 在预算有限时，策略4.2的效果通常更好

## 策略4.3：整数线性规划（ILP）

### 设计起因
贪心算法和分层策略都是近似解法，可能错过全局最优。整数线性规划可以找到**精确最优解**——在预算约束下，覆盖权重之和最大的组合子集。

### 目的
在候选组合数量可控时（<1000），找到数学上的最优购买方案。

### 逻辑

```
输入：
  candidates = 候选组合列表，共 n 组
  number_weights = 号码权重表，共 m 个候选号码
  budget = 用户预算

建模：
  决策变量：
    xᵢ ∈ {0, 1}  # 第i组是否购买（i=1..n）
    yⱼ ∈ {0, 1}  # 第j个号码是否被覆盖（j=1..m）

  目标函数：
    maximize Σ(wⱼ * yⱼ)  for j=1..m
    含义：最大化被覆盖号码的权重之和

  约束条件：
    Σ(xᵢ) * 2 ≤ budget           # 预算约束
    yⱼ ≤ Σ(xᵢ : j ∈ sᵢ)         # 覆盖约束：号码j被覆盖当且仅当至少一组包含j的组合被购买
    xᵢ ∈ {0,1}                    # 整数约束
    yⱼ ∈ {0,1}                    # 整数约束

求解：
  使用 PuLP 或 Google OR-Tools 求解器
  候选组合 < 200 时，通常在秒级内求解完成

输出：
  精确最优的购买方案
```

### 优缺点
- 优点：数学上的精确最优解
- 缺点：候选组合过多时求解时间可能较长
- 适用场景：候选组合数量 < 1000 时使用（本项目完全满足）

## 策略4.4：复式/胆拖投注优化

### 设计起因
前三个策略都是在"单式投注"框架下优化——每注2元，选一组号码。但彩票还有**复式**和**胆拖**两种玩法，它们可以用一张票覆盖多注组合，在某些情况下比买多张单式更划算。

### 目的
利用复式和胆拖玩法的数学特性，找到"花更少钱覆盖更多号码"的方案。

### 复式投注规则

```
大乐透复式：
  前区：可选 6~20 个号码（标准是5个）
  后区：可选 3~12 个号码（标准是2个）
  注数 = C(前区选号数, 5) × C(后区选号数, 2)
  价格 = 注数 × 2元

  示例：
    前区选7个，后区选3个：
    注数 = C(7,5) × C(3,2) = 21 × 3 = 63注
    价格 = 63 × 2 = 126元

    前区选6个，后区选3个：
    注数 = C(6,5) × C(3,2) = 6 × 3 = 18注
    价格 = 18 × 2 = 36元

双色球复式：
  红球：可选 7~20 个号码（标准是6个）
  蓝球：可选 2~16 个号码（标准是1个）
  注数 = C(红球选号数, 6) × C(蓝球选号数, 1)
  价格 = 注数 × 2元
```

### 胆拖投注规则

```
大乐透胆拖：
  胆码：每注必含的号码（前区最多4个，后区最多1个）
  拖码：备选号码，从中选取补满
  注数 = C(前区拖码数, 5-前区胆码数) × C(后区拖码数, 2-后区胆码数)

  示例：
    前区胆码2个，拖码5个；后区胆码1个，拖码2个：
    注数 = C(5, 5-2) × C(2, 2-1) = C(5,3) × C(2,1) = 10 × 2 = 20注
    价格 = 20 × 2 = 40元
    覆盖号码：前区 2+5=7个，后区 1+2=3个

  对比单式：
    覆盖同样7个前区号码和3个后区号码
    如果买单式需要 C(7,5)×C(3,2) = 63注 = 126元
    胆拖只需 20注 = 40元（因为胆码每注必含，减少了组合数）
```

### 优化逻辑

```
输入：
  number_weights = 号码权重表
  top_numbers = 按权重排序的号码列表
  budget = 用户预算

处理：
  步骤1：确定胆码候选
    前区胆码候选 = 权重最高的 1~4 个前区号码
    后区胆码候选 = 权重最高的 1 个后区号码
    胆码是"最有信心"的号码，每注必含

  步骤2：枚举胆码组合
    前区胆码数量从1到4枚举
    后区胆码数量从0到1枚举
    （胆码数量少，枚举空间小，完全可行）

  步骤3：对每种胆码组合，贪心选择拖码
    按权重降序，依次将号码加入拖码
    每加一个拖码，计算新增注数和花费
    当花费即将超过预算时停止

  步骤4：计算每种方案的性价比
    对于每种 胆码+拖码 方案：
      total_cost = 注数 × 2
      covered_weight = 所有覆盖号码的权重之和
      efficiency = covered_weight / total_cost

  步骤5：选择性价比最高的方案
    best_plan = argmax(efficiency)
    如果 best_plan 的性价比优于纯单式方案 → 采用胆拖
    否则 → 采用单式

输出：
  最优的胆拖/复式方案（或单式方案）
```

### 胆拖 vs 复式 vs 单式的选择

```
决策逻辑：
  如果预算 < 10元：
    只能买单式（复式/胆拖最低花费通常 > 10元）

  如果预算 10~50元：
    比较胆拖方案和单式方案的性价比
    通常胆拖更优（因为胆码减少了组合数）

  如果预算 50~200元：
    比较复式、胆拖、单式三种方案
    可能采用混合方案（一张胆拖 + 几张单式）

  如果预算 > 200元：
    可以考虑多张复式/胆拖的组合
```

## 购买方案评估函数

### 设计起因
四种策略各自产出一个购买方案，需要一个统一的评估函数来比较它们，选出最优方案。

### 目的
量化评估每个购买方案的综合质量。

### 逻辑

```
输入：
  plan = 一个购买方案（包含所有票据信息）
  number_weights = 号码权重表

处理：
  指标1：总花费
    total_cost = 所有票据的花费之和
    （越低越好，但不是唯一目标）

  指标2：总注数
    total_combinations = 所有票据展开后的注数之和
    （越多越好，意味着覆盖面越广）

  指标3：覆盖号码数
    unique_numbers = 方案中出现的所有不同号码
    （越多越好）

  指标4：覆盖权重
    weight_coverage = sum(number_weights[n] for n in unique_numbers)
    （越高越好，核心指标）

  指标5：性价比
    cost_efficiency = weight_coverage / total_cost
    （每元钱覆盖的权重值，越高越好）

  指标6：Top号码覆盖率
    top10_coverage = 权重Top10号码中被覆盖的个数 / 10
    （越高越好，确保最重要的号码被覆盖）

  指标7：多样性
    diversity = 方案中各组合之间的平均差异度
    （越高越好，避免买了一堆相似的组合）

  综合评分：
    final_score = 0.30 * normalize(cost_efficiency)
               + 0.25 * normalize(weight_coverage)
               + 0.20 * normalize(top10_coverage)
               + 0.15 * normalize(total_combinations)
               + 0.10 * normalize(diversity)

输出：
  方案的综合评分 + 各项指标明细
```

## 四种策略的协作与最终决策

```
策略4.1（贪心覆盖）────→ 方案A
策略4.2（分层覆盖）────→ 方案B
策略4.3（ILP精确解）───→ 方案C
策略4.4（复式/胆拖）───→ 方案D

        ↓ 全部用评估函数打分 ↓

  方案A: score=0.72
  方案B: score=0.78
  方案C: score=0.81  ← 最优单式方案
  方案D: score=0.85  ← 最优胆拖方案

        ↓ 选择得分最高的 ↓

  最终方案 = 方案D（胆拖）

  或者：混合方案 = 方案D的胆拖部分 + 方案C的补充单式
  （如果预算允许且混合方案得分更高）
```

## 与整条管道的关系总结

```
阶段0 提供数据基座 → 所有阶段共享
阶段1 缩小候选池   → 排除30%-50%号码
阶段2 量化优先级   → 每个号码一个权重分数
阶段3 生成候选组合 → 50-200组高质量组合
阶段4 优化购买方案 → 一个可执行的购买方案 ← 最终产出

每个阶段都不可缺少：
  没有阶段0 → 后续算法无法获取特征
  没有阶段1 → 权重引擎噪声大，精度低
  没有阶段2 → 组合生成只能随机选，无法区分优劣
  没有阶段3 → 没有候选组合可供优化
  没有阶段4 → 用户拿到一堆组合不知道怎么买
```
