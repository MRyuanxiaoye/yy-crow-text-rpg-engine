# 阶段3：组合生成器（Combination Generator）

## 在方法链中的位置

```
  阶段0：数据预处理与特征工程
        ↓
  阶段1：排除引擎
        ↓
  阶段2：权重引擎
        ↓
【阶段3：组合生成器】 ← 你在这里
        ↓
  阶段4：购买优化器
```

## 为什么是第四步？

### 起因
经过阶段1和阶段2，我们已经有了：
- 一个缩小后的候选号码池（排除了30%-50%的号码）
- 每个候选号码的权重分数（量化了出现可能性）

但这些还只是**单个号码**的信息。彩票买的是**号码组合**——大乐透要选5+2，双色球要选6+1。一组好的号码组合不仅要求每个号码权重高，还要求组合整体满足各种约束（奇偶均衡、和值合理、不扎堆等）。

### 目的
将单个号码的权重信息转化为**多组完整的候选号码组合**，每组组合都满足彩票规则和历史统计约束，并按综合得分排序。

### 为什么不能直接取权重最高的号码？
假设直接取权重Top5作为大乐透前区号码，可能出现：
- 5个号码全是奇数（历史上全奇概率<2%）
- 5个号码集中在同一区间（如全在1-12之间）
- 和值极端偏高或偏低
- 这样的组合虽然每个号码权重都高，但组合整体不合理

组合生成器的价值就在于：**在高权重号码中，找到组合整体也合理的方案**。

## 硬约束与软约束

### 硬约束（必须满足，否则无法购买）

```
大乐透：
  前区：从1-35中选5个不重复号码，升序排列
  后区：从1-12中选2个不重复号码，升序排列
  单注价格：2元

双色球：
  红球：从1-33中选6个不重复号码，升序排列
  蓝球：从1-16中选1个号码
  单注价格：2元
```

### 软约束（基于历史统计，不满足也能买，但概率极低）

```
这些约束的参数全部从历史数据动态计算，不硬编码。

奇偶比约束：
  历史上最常见的奇偶比（以大乐透前区5个号码为例）：
    3:2 和 2:3 合计占比约60%
    4:1 和 1:4 合计占比约29%
    5:0 和 0:5 合计占比约3%（极端，应避免）

大小比约束：
  以号码范围中位数为界，大号和小号的比例
  最常见：3:2 和 2:3，合计约55%

和值约束：
  所有号码之和应落在历史分布的95%置信区间内
  如大乐透前区和值通常在 [55, 120] 之间

跨度约束：
  最大号 - 最小号，应落在历史90%分位区间内
  如大乐透前区跨度通常在 [18, 34] 之间

连号约束：
  相邻号码（如5和6）的组数，历史上0-2组占95%以上

AC值约束：
  AC值 = 号码两两之差的去重个数 - (选号个数-1)
  历史上AC值 ≥ 4 的占90%以上
  AC值越高，号码越"分散"，组合越合理
```

## 策略3.1：加权随机采样

### 设计起因
最直接的方法：把号码权重当作概率分布，按概率抽样。权重高的号码被抽中的概率大，权重低的也有机会。这种方法天然产生多样性——每次抽样结果不同。

### 目的
快速生成大量候选组合，作为后续筛选和优化的原材料。

### 逻辑

```
输入：
  number_weights = 阶段2输出的号码权重表（和为1的概率分布）
  num_to_select = 需要选取的号码个数（大乐透前区=5）
  num_samples = 需要生成的组合数量（建议M=5000）
  soft_constraints = 软约束检查函数

处理：
  candidates = []
  attempts = 0

  while len(candidates) < num_samples and attempts < num_samples * 10:
    attempts += 1

    步骤1：按权重无放回采样
      selected = 从候选号码中按 number_weights 概率无放回抽取 num_to_select 个
      selected.sort()  # 升序排列

    步骤2：检查软约束
      如果 check_soft_constraints(selected) 通过:
        candidates.append(selected)
      否则:
        丢弃，重新采样

  return candidates

输出：
  满足软约束的候选组合列表（最多 num_samples 组）
```

### 优缺点
- 优点：实现简单，速度快，天然多样性
- 缺点：软约束通过率可能不高（尤其约束严格时），大量采样被丢弃
- 适用场景：作为初始候选池的生成方法，为遗传算法提供初始种群

## 策略3.2：贪心构造 + 约束修正

### 设计起因
加权随机采样的问题是"碰运气"——生成的组合可能大量不满足约束。贪心构造的思路是：**逐个选入号码，每选一个就检查约束**，如果违反就跳过。这样保证每组生成的组合都满足约束。

### 目的
生成一批**确定满足所有软约束**的高质量组合。

### 逻辑

```
输入：
  number_weights = 号码权重表
  num_to_select = 需要选取的号码个数
  num_combos = 需要生成的组合数量（建议50~100）

处理：
  对于每组组合：
    selected = []
    remaining = 所有候选号码的副本

    步骤1：引入随机扰动
      对权重添加小幅随机噪声，确保每次生成不同的组合
      perturbed_weights[v] = number_weights[v] * (1 + random(-0.2, 0.2))

    步骤2：按扰动后的权重降序排列候选号码
      sorted_candidates = sort(remaining, key=perturbed_weights, descending)

    步骤3：逐个选入
      for v in sorted_candidates:
        假设将 v 加入 selected
        检查加入后是否违反任何软约束：
          - 奇偶比是否会走向极端？
          - 大小比是否会走向极端？
          - 和值是否会超出范围？
          - 是否产生过多连号？

        如果不违反：
          selected.append(v)
        如果违反：
          跳过 v，尝试下一个

        如果 len(selected) == num_to_select:
          break

    步骤4：验证完整性
      如果 len(selected) < num_to_select:
        约束过严导致选不满，放宽约束重试
      否则:
        加入候选列表

输出：
  确定满足软约束的候选组合列表
```

### 优缺点
- 优点：每组都满足约束，质量有保证
- 缺点：多样性不足（即使加了随机扰动，组合仍然相似）
- 适用场景：生成"保底"的高质量组合，确保候选池中有足够多的合理组合

## 策略3.3：遗传算法生成

### 设计起因
策略3.1（随机采样）有多样性但质量不稳定，策略3.2（贪心构造）有质量但多样性不足。遗传算法结合了两者的优点：通过**进化机制**，在保持多样性的同时逐步提升组合质量。

### 目的
通过模拟自然选择，进化出一批**既高质量又多样化**的候选组合。

### 逻辑

```
输入：
  number_weights = 号码权重表
  combo_stats = 组合特征的历史统计
  population_size = 种群大小（建议P=200）
  generations = 迭代代数（建议G=100）

处理：
  步骤1：初始化种群
    用策略3.1（加权随机采样）生成 P 组候选作为初始种群
    population = [combo₁, combo₂, ..., comboₚ]

  步骤2：适应度评估
    对每个组合计算适应度分数：
      fitness(combo) = w₁ * weight_score(combo)
                     + w₂ * constraint_score(combo)
                     + w₃ * diversity_score(combo)

    其中：
      weight_score = 组合中所有号码权重之和（越高越好）
      constraint_score = 满足软约束的程度（0~1）
      diversity_score = 与种群中其他组合的平均差异度（越高越好）

    建议权重：w₁=0.5, w₂=0.3, w₃=0.2

  步骤3：选择（锦标赛选择）
    每次随机取 k=3 个组合，选适应度最高的作为父代
    重复直到选出足够的父代对

  步骤4：交叉（均匀交叉）
    对于两个父代 parent_A 和 parent_B：
      合并两个父代的所有号码 → union_set
      从 union_set 中按权重采样选出 num_to_select 个号码
      生成子代 child

    示例（大乐透前区）：
      parent_A = [3, 8, 14, 25, 31]
      parent_B = [5, 8, 19, 25, 33]
      union_set = [3, 5, 8, 14, 19, 25, 31, 33]
      child = 从中按权重选5个，如 [5, 8, 14, 25, 33]

  步骤5：变异
    以变异概率 p_mut=0.1 对子代进行变异：
      随机替换1个号码为候选池中的其他号码（按权重选择替换号码）

  步骤6：精英保留
    将当前代适应度最高的 top 10% 直接保留到下一代
    其余位置由子代填充

  步骤7：重复步骤2-6，迭代 G 代

  步骤8：输出最终种群中适应度最高的 N 组

输出：
  进化后的高质量候选组合列表
```

### 多样性保护机制
- 适应度函数中包含 diversity_score，鼓励与已有组合不同
- 锦标赛选择（而非轮盘赌）保留了弱个体的生存机会
- 变异操作引入新的号码，防止种群收敛到局部最优

### 优缺点
- 优点：兼顾质量和多样性，全局搜索能力强
- 缺点：计算量较大（P*G次适应度评估），但M3可以胜任
- 适用场景：作为主力生成策略，产出最终候选组合的主体

## 策略3.4：蒙特卡洛树搜索（MCTS）

### 设计起因
遗传算法是在"完整组合"层面做搜索，而MCTS是在"逐个选号"层面做搜索。它把选号过程建模为一棵树：根节点是空集，每一层选入一个号码，叶节点是完整组合。通过大量模拟，MCTS能在巨大的搜索空间中找到高质量的路径。

### 目的
通过树搜索，发现遗传算法可能遗漏的高质量组合，作为补充。

### 逻辑

```
输入：
  number_weights = 号码权重表
  num_to_select = 需要选取的号码个数
  num_simulations = 模拟次数（建议1000~5000）

处理：
  树结构：
    节点 = {已选号码集合, 访问次数N, 累计价值Q}
    根节点 = {已选: 空集, N: 0, Q: 0}

  重复 num_simulations 次：

    步骤1：选择（Selection）
      从根节点出发，用 UCB1 公式选择子节点：
        UCB1(child) = Q(child)/N(child) + C * sqrt(ln(N(parent)) / N(child))
        其中 C 是探索系数（建议C=1.414）
      一直选到叶节点（未完全展开的节点）

    步骤2：扩展（Expansion）
      在叶节点上，选择一个未尝试过的号码作为新子节点
      选择策略：按号码权重概率选择（偏向高权重号码）

    步骤3：模拟（Simulation）
      从新节点出发，随机补全剩余号码（按权重采样）
      得到一个完整组合

    步骤4：评估（Evaluation）
      用组合评分函数计算该组合的得分 score

    步骤5：回传（Backpropagation）
      沿路径回传，更新每个节点的 N 和 Q：
        N(node) += 1
        Q(node) += score

  最终输出：
    从根节点出发，每层选择 Q/N 最高的子节点
    得到最优路径对应的组合
    同时收集模拟过程中得分最高的 Top-N 组合

输出：
  MCTS发现的高质量候选组合列表
```

### 优缺点
- 优点：在大搜索空间中高效探索，UCB1平衡了利用与探索
- 缺点：实现复杂度较高，单次搜索只产出少量组合
- 适用场景：作为遗传算法的补充，发现不同类型的高质量组合

### 与遗传算法的互补
- 遗传算法：在"组合空间"中全局搜索，产出大量多样化组合
- MCTS：在"选号决策树"中深度搜索，产出少量但可能更优的组合
- 两者的候选合并后，覆盖面更广

## 组合评分函数

### 设计起因
无论用哪种生成策略，都需要一个统一的评分函数来衡量"这组号码有多好"。评分函数是所有策略的共同基础。

### 目的
为每组候选号码计算一个综合得分（0~1），用于排序和筛选。

### 逻辑

```
输入：
  combo = 一组号码（如 [3, 8, 14, 25, 31]）
  number_weights = 号码权重表
  combo_stats = 历史组合特征统计

处理：
  子分数1：权重得分（占比40%）
    weight_score = sum(number_weights[n] for n in combo)
    归一化：除以理论最大值（Top-K号码权重之和）
    含义：组合中号码的权重越高越好

  子分数2：约束满足度（占比25%）
    检查每个软约束的满足程度：
      odd_even_ok = 奇偶比是否在常见范围内？（0或1）
      size_ratio_ok = 大小比是否在常见范围内？（0或1）
      sum_ok = 和值是否在95%置信区间内？（0或1）
      span_ok = 跨度是否在90%分位区间内？（0或1）
      consecutive_ok = 连号组数是否 ≤ 2？（0或1）
      ac_ok = AC值是否 ≥ 4？（0或1）

    constraint_score = 满足的约束数 / 总约束数
    含义：满足的约束越多越好

  子分数3：均衡性得分（占比20%）
    将号码范围分为3个区间，统计每个区间的号码个数
    理想分布：每个区间都有号码
    balance_score = 1 - 标准差(各区间号码数) / 最大可能标准差
    含义：号码分布越均匀越好，避免扎堆

  子分数4：历史相似度（占比15%）
    计算该组合与历史中奖组合的平均余弦相似度
    取最相似的 Top-5 历史组合的平均相似度
    similarity_score = mean(top5_similarities)
    含义：与历史中奖组合越相似越好（但不能完全相同）

  综合得分：
    final_score = 0.40 * weight_score
               + 0.25 * constraint_score
               + 0.20 * balance_score
               + 0.15 * similarity_score

输出：
  combo 的综合得分（0~1之间）
```

### 评分权重的设计理由
- **权重得分占40%**：号码权重是整条管道的核心产出，应该是最重要的因素
- **约束满足度占25%**：不满足约束的组合在历史上极少出现，必须重视
- **均衡性占20%**：号码分布均匀是历史中奖组合的普遍特征
- **历史相似度占15%**：作为辅助参考，但不宜过高（避免过拟合历史数据）

## 与上下游的衔接

### 从阶段2接收
- `number_weights`：每个候选号码的权重（作为采样概率和评分依据）
- `weight_details`：权重明细（用于分析和可视化）

### 向阶段4传递
- `candidates`：按得分排序的候选组合列表（建议50~200组）
- 每组包含：号码、得分、得分明细
- 阶段4的购买优化器将从这些候选中选择最优购买方案

### 四种策略的协作方式
```
策略3.1（随机采样）──→ 生成5000组原始候选
                          ↓
                     过滤掉不满足软约束的
                          ↓
策略3.2（贪心构造）──→ 补充50组保底高质量组合
                          ↓
                     合并为遗传算法初始种群
                          ↓
策略3.3（遗传算法）──→ 进化100代，输出Top100
                          ↓
策略3.4（MCTS）─────→ 补充20组深度搜索组合
                          ↓
                     合并去重，按评分排序
                          ↓
                     输出最终 Top 50~200 组候选
```
