# 核心研究原则

## 科学方法论优先

**目标不是"预测号码"，而是"做好研究"。** 只要每一步都走得足够科学、研究得足够透彻，那么得到的必然是足够接近准确的预测。准确的预测是科学研究的自然产物，而非直接追求的目标。

## "量大管饱"实验基础标准

**这是所有实验计划和代码生成的最高优先级标准。**

**核心原则**：每一条挖掘出的规则，不论最终是否有效，都必须经过严格回测验证并得出有事实依据的结论。

**具体要求**：

1. **规则零遗漏**——所有已挖掘的规则类型（A1/A2/A3/A4/B1）、所有彩种（双色球/大乐透）、所有号码区域（红球/蓝球），都必须纳入实验验证。任何规则不得因"可能没用"或"优先级低"而跳过。
2. **结论必须基于事实**——每条规则/每类规则的结论（有效/无效/增量贡献多少）必须有严格时间切分的回测数据支撑。"没用"也是有价值的结论，因为排除了不需要的方向。
3. **不做选择性验证**——不能只验证"看起来有希望的"规则而忽略其他。实验计划必须覆盖所有规则类型的完整验证。
4. **补全优先于推进**——如果发现有规则类型未被验证（如 A3/A4 未纳入模型、蓝球未挖掘），必须先补全验证，再推进后续实验。补全是基础，基础不牢地动山摇。
5. **严格版规则同步**——如果全量版规则已挖掘但严格版（前60%数据）未挖掘，必须补挖严格版，确保所有结论都基于无数据泄露的验证。

## 约束条件必须有事实依据

1. **禁止拍脑袋定参数**——每一个阈值、每一个约束条件，都必须有数据支撑或理论依据。如果没有，就先做实验验证。
2. **避免一刀切**——不同局面、不同位置、不同置信度下，应该有不同的处理策略。硬编码的统一阈值是懒惰的做法。
3. **保留信息，延迟决策**——模型输出的连续概率/权重应尽可能保留到最后一步，而不是过早地二值化（排除/保留）。排除是权重为零的极端情况，应由数据驱动决定何时可以安全排除。

## 规则适配是核心

核心不是"用规则排除号码"，而是"找到最适配当前局面的规则，然后根据学习验证出的规则结果分配号码权重"。具体而言：
- 规则的作用是提供信号，而非直接做硬排除
- 不同规则在不同局面下的可靠性不同，模型应学习这种适配关系
- 最终输出应该是每个号码的权重/概率分布，而非二值的"保留/排除"

## 多维度关联思维

数据之间的联系不是单维度的，必须从多个维度去建模和验证：

### 需要考虑的关联维度

1. **同期组内关联**——同一期的 P0-P4 之间：
   - 排序约束（P0 < P1 < ... < P4）
   - 间距分布（相邻位置的差值有统计规律）
   - 组合统计量（和值、跨度、奇偶比、大小比、AC值）
   - 号码之间的互斥/共现关系

2. **跨期同位置关联**——不同期的同一位置之间：
   - 方向惯性（连续上升/下降的趋势）
   - 值域回归（偏离均值后的回归倾向）
   - 周期性模式

3. **跨期跨位置关联**——不同期、不同位置之间：
   - 相邻位置的联动（P2 上升时 P3 的行为）
   - 整体走势的协同（全体偏大/偏小的趋势）
   - 差分的跨位置传播

4. **组合层面的全局约束**——不是单个号码的属性，而是整组号码的属性：
   - 和值范围、奇偶比、大小比等在历史上的分布
   - 这些约束在组合层面过滤，比逐位置过滤更有效

### 当前方案的局限

当前 phase3 是逐位置独立训练分类器，完全忽略了上述关联。这导致：
- 各位置预测可能相互矛盾（如预测 P2 上升到 20，同时预测 P3 下降到 18）
- 无法利用组合层面的约束信息
- 位置间的协同信息被浪费

后续实验应逐步引入这些关联维度，验证每个维度的增量贡献。

## E1 暴露的教训

E1 中 `概率 < 0.15 则排除` 的做法违反了上述原则：
- 0.15 没有数据依据
- 所有位置、所有期数一刀切
- 将连续概率硬切为二值，丢失了大量信息
- 导致组合存活率从位置级 93% 降至 69-71%

后续实验应改为权重化方案：模型输出概率直接作为号码权重，按权重排序/筛选，阈值由数据驱动确定。

# RichRichRich 项目架构计划

## 当前架构：v2 - 分层缩号架构

**状态**：待实现
**确定时间**：2026-02-07

### 核心思路

全号码空间 → 走势过滤 → 区间锁定 → 规则排除 → 最终候选号码

不再追求单条规则的高准确率，而是通过多层过滤逐步缩减候选号码数量。规则不是一条，而是一群，且每条规则有各自的适用条件，系统需要自动判断当前局面下哪些规则被激活。

### 架构流程

```
输入：当前最新一期开奖号码 + 历史数据

Step 1: 走势预测层
  → 每个位置输出方向概率 (升/降/平)
  → 锁定候选区间
  → 不做硬切，输出概率（如：升72%、降20%、平8%）
  → 概率不够高时保留两个方向的候选，给低概率方向较低权重

Step 2: 约束传播层
  → 位置间排序约束（P0 < P1 < P2 < P3 < P4）
  → 相邻位置联动收紧区间
  → 例：P2预测升(>12)，P3预测降(<20)，则P2候选[13,19]，P3候选[P2+1,19]

Step 3: 规则排除层
  → 扫描所有规则，匹配当前局面
  → 激活的规则对候选号码投票排除
  → 条件匹配 + 置信度加权
  → 多条规则对同一号码的排除/保留投票加权汇总
  → 超阈值排除

Step 4: 输出
  → 每个位置的最终候选号码列表
  → 每个排除决策的规则依据

Step 5: 回测验证（流水线回测）
  → 在历史数据上跑完整流水线
  → 评估指标：
    - 缩减率：平均每个位置排除了多少比例的号码
    - 存活率：正确号码没被误排除的比例
    - 综合效率：缩减率 × 存活率
```

### 规则来源（多元化）

| 模块 | 规则类型 | 作用层 |
|------|----------|--------|
| A1 方向模式 | 走势方向预测 | Step 1 走势预测层（主力） |
| A2 条件规则 | 多条件组合排除 | Step 3 规则排除层（主力） |
| A3 差分模式 | 差分值域约束 | Step 3 辅助缩小区间 |
| A4 统计量模式 | 和值/跨度/AC值等全局约束 | Step 3 从整体角度排除不合理组合 |
| B1 Transformer | 端到端预测 | 独立投票者，参与各层 |

### 关键设计原则

1. 走势预测输出概率而非硬判断，避免误排除
2. 规则适配是条件匹配+置信度加权，不是二分有效/无效
3. 位置间存在排序约束，约束传播可进一步缩小空间
4. 回测方式是完整流水线回测，而非单条规则验证

### v2 前置实验：规则适配学习实验

**状态**：已完成（含严格时间切分验证）
**完成时间**：2026-02-08
**目的**：验证"让模型学习规则适配"的可行性，为 v2 Step 3 规则排除层提供数据支撑和方案选型

#### 严格验证结论（时间切分：前60%挖掘规则，后40%训练+测试）

| 彩种 | 最优方案 | 方向准确率 | 缩减率 | 存活率 | 综合效率 |
|------|---------|-----------|--------|--------|---------|
| 大乐透 | 3B XGBoost | 85.1% | 28.1% | 93.0% | 26.2% |
| 双色球 | 3C MLP | 86.7% | 35.4% | 93.1% | 32.9% |

与原始实验（全量数据挖掘规则）对比：
- 大乐透：效率下降 5.87%（轻度过拟合，仍 >= 0.25 可行阈值）
- 双色球：效率反而提升 4.80%（无过拟合，精简规则效果更优）
- 方向预测能力稳定在 85%+，不受数据泄露影响

特征重要性：局面特征（上期值等）~50%，规则特征 ~30%，元信息 ~20%

结果文件：`src/research/results/experiment_strict/real_comparison.json`

#### 实验背景

v2 架构的核心难点在 Step 3 规则排除层：2585万条规则（A2），加上 A1/A3/A4 的数万条模式，如何在给定局面下自动选择合适的规则子集来排除号码？不能靠人工写适配逻辑，需要让模型从历史数据中学习"什么局面下用什么规则"。

#### 实验设计

##### 阶段一：规则压缩（解决规则数量爆炸问题）

**目标**：将 2585万条 A2 规则压缩到可管理规模

**方法**：
1. 条件特征化：将每条规则的 conditions 编码为特征向量
   - 条件类型：val/dir/odd/even/big/small/diff/sum/span/ac/consec（11类）
   - 涉及位置：P0-P4（5位）
   - target 方向：U/D/E（3类）
   - target 位置：P0-P4
2. 聚类：对特征向量做层次聚类或 K-Means
   - 目标簇数：500-2000（通过轮廓系数选择）
   - 同簇规则共享相似的条件结构和预测目标
3. 簇代表规则：每个簇选 confidence 最高、support 最大的规则作为代表
4. 对 A1（21,248条）、A3（6,117条）、A4（14,634条）做同样处理，但这些数量较小，可能不需要聚类，直接按 confidence 筛选 top 规则即可

**输出**：规则簇表（cluster_id → 代表规则 + 簇内规则数 + 平均 confidence/lift）

**验证**：压缩前后在 C1 回测上的准确率损失 < 5%

**重要说明**：压缩是实验手段，不一定用于最终方案。
- 如果压缩后效果好（损失<5%）→ 最终方案可保留压缩，推理速度快
- 如果压缩损失明显 → 最终方案不压缩，改为"先触发扫描再处理"：每期从2585万条中筛出被触发的几千条，直接用这些规则参与排除
- 实验阶段必须压缩的原因是需要构造固定维度的特征向量来训练模型

##### 阶段二：历史回放（构造训练数据）

**目标**：对每一期历史数据，记录所有规则的触发状态和预测结果

**方法**：
1. 时间切分（严格防止数据泄露）：
   ```
   大乐透 2833 期：
     规则挖掘期：第 1-1700 期（60%）
     适配训练期：第 1701-2400 期（25%）
     最终测试期：第 2401-2833 期（15%）

   双色球 3413 期：
     规则挖掘期：第 1-2050 期（60%）
     适配训练期：第 2051-2900 期（25%）
     最终测试期：第 2901-3413 期（15%）
   ```
   注意：现有规则是用全量数据挖掘的，实验需要用规则挖掘期重新挖掘规则

2. 对适配训练期的每一期，生成一条样本：
   ```
   特征（X）：
     - 当前局面特征：最近 N 期号码、走势方向、差分值、统计量
     - 各规则簇触发状态：0/1 向量（该簇的代表规则条件是否满足）
     - 触发规则的元信息：触发簇数量、平均 confidence、平均 lift

   标签（Y）：按位置拆分
     - 对每个位置的每个候选号码：是否为正确开奖号码（0/1）
     - 或简化为：每个位置的正确号码值
   ```

3. 样本展开方式：
   - 按位置拆分：每期 × 5个位置（大乐透）= 5条样本
   - 按候选号码展开：每个位置 × 候选号码数 = 更多样本（用于排除模型）

**输出**：训练数据表（CSV/Parquet），包含特征矩阵和标签

##### 阶段三：规则适配模型训练

**目标**：训练模型学习"当前局面 → 哪些规则有效 → 排除哪些号码"

**方法**：分三个子实验，从简单到复杂

**实验 3A：基线 - 简单统计适配**
- 不用机器学习，纯统计方法
- 对每个规则簇，计算其在适配训练期的准确率
- 只激活历史准确率 > 阈值的规则簇
- 评估：缩减率 + 存活率

**实验 3B：梯度提升树（XGBoost/LightGBM）**
- 输入：当前局面特征 + 规则簇触发向量
- 输出：每个候选号码的排除概率
- 优势：天然处理稀疏特征、条件组合，训练快，可解释（feature importance 可以看出哪些规则簇最有用）
- 超参搜索：max_depth, n_estimators, learning_rate, min_child_weight

**实验 3C：轻量神经网络**
- 输入同 3B
- 模型：2-3层 MLP，或小型 Transformer
- 对比 3B 是否有提升
- 如果 3B 已经足够好，不需要上 3C

**评估指标**（统一）：
```
对测试期每一期：
  1. 用走势预测（A1/B1）锁定候选区间
  2. 用适配模型排除号码
  3. 记录：
     - 缩减率 = 1 - (剩余候选数 / 区间内总候选数)
     - 存活率 = 正确号码在剩余候选中的比例
     - 综合效率 = 缩减率 × 存活率
     - 每个位置分别统计
```

**目标基线**：
- 缩减率 > 30%（至少排除三成候选号码）
- 存活率 > 85%（正确号码不被误排除的概率）
- 综合效率 > 0.25

##### 阶段四：结果分析

1. 对比 3A/3B/3C 的缩减率和存活率
2. 分析 feature importance：哪些规则簇/局面特征对排除最有贡献
3. 分析失败案例：正确号码被误排除时，是哪些规则导致的
4. 决定 v2 Step 3 采用哪种适配方案
5. 确定是否需要回到阶段一调整规则压缩策略

#### 实验执行顺序

```
阶段一（规则压缩）
  ↓
阶段二（历史回放 + 数据构造）
  ↓
实验 3A（统计基线）→ 评估
  ↓
实验 3B（XGBoost）→ 评估 → 对比 3A
  ↓
（如果 3B 不够好）实验 3C（神经网络）→ 评估
  ↓
阶段四（结果分析 + 方案决策）
```

#### 关键风险

1. **数据泄露**：现有规则用全量数据挖掘，实验必须用切分后的训练期重新挖掘
2. **过拟合**：规则本身就是数据挖掘产物，再训练适配模型是二次拟合，需严格时间切分
3. **样本量不足**：适配训练期只有 700/850 期，展开后也只有几千条样本
4. **规则压缩损失**：聚类可能丢失关键规则，需要验证压缩前后效果差异

#### 预期产出

1. 规则簇表（压缩后的规则库）
2. 历史回放数据集

## 研究结论（截至 2026-02-08）

### 核心发现

**方向预测能力是真实的，缩号幅度有限但可行。**

通过"穷举挖掘统计规则 → 聚类压缩 → 机器学习适配选择"的三级管线，在严格时间切分条件下实现了**每个位置约 30% 的候选号码缩减，存活率 93%**。方向预测是核心驱动力，规则特征贡献约 30% 的预测能力，局面特征（上期值、差分）贡献约 50%。

### 各方法的方向预测能力

| 方法 | 方向准确率 | 基线（随机） | 备注 |
|------|-----------|-------------|------|
| A1 规则回测 | 60-62% | 33.3% | 覆盖率太低（<16%） |
| Transformer | 47% | 33.3% | 单独使用效果有限 |
| XGBoost 规则适配 | 85.1% | 33.3% | 大乐透最优 |
| MLP 规则适配 | 86.7% | 33.3% | 双色球最优 |

### 严格验证后的缩号效率

| 彩种 | 综合效率 | 缩减率 | 存活率 | 最优方案 |
|------|---------|--------|--------|---------|
| 大乐透 | 26.2% | 28.1% | 93.0% | XGBoost |
| 双色球 | 32.9% | 35.4% | 93.1% | MLP |

### 不成立的假设

- A1 规则单独使用覆盖率太低（<16%），大部分期数无法触发
- Transformer 单独使用效果有限（47%），远不如规则适配方案
- 规则聚类 silhouette score 很低（0.11），规则空间无自然簇结构，聚类仅为降维手段

### E1 组合级评估结论

**位置级缩减不等于组合级实用。** 虽然每位置缩减 28-35%，组合层面缩减达 62-75%（乘法效应），但组合存活率从位置级 93% 降至 69-71%（任一位置误排除即失败）。平均仍需 12-27 万注覆盖，每期成本 24-55 万元，当前方案不具备实用价值。需要 E2 多层级联过滤或 E3 蓝球预测进一步提升。

| 指标 | 大乐透 | 双色球 |
|------|--------|--------|
| 组合缩减率 | 62.6% | 75.2% |
| 组合存活率 | 70.6% | 68.8% |
| 概率提升倍数 | 2.67x | 4.03x |
| 平均需购注数 | 121,427 | 274,553 |

### E1.5 多维权重化评估结论

**三个关键判断：**

1. **瓶颈不在阈值选择，而在信息量。** 权重化方案（基于差分条件分布）的 AUC 为 0.78（大乐透）/ 0.83（双色球），排序能力真实有效——正确号码平均排在前 20-27%。但权重化 vs 硬阈值的组合效率持平（双色球 0.519 vs 0.517），说明换更科学的阈值方式并不能突破单层方向预测的信息量上限。

2. **组合统计约束的净增益很小。** 和值、奇偶比、间距等约束叠加可过滤 30% 的随机组合，但正确组合仅 65-67% 通过。过滤率和存活率几乎抵消，不应作为 E2 的主力方向。

3. **跨位置方向关联是最大的未利用信息源。** 所有相邻位置对的方向变化高度相关（KL 散度 0.11-0.19，χ² p≈0）。条件预测准确率 72% vs 独立预测 47%，提升 52%。当前模型逐位置独立预测，完全浪费了这个信息。

**量化数据：**

| 指标 | 大乐透 | 双色球 |
|------|--------|--------|
| 号码排序 AUC | 0.782 | 0.833 |
| 权重化 K95 组合效率 | 0.326 | 0.519 |
| E1 硬阈值组合效率 | 0.442 | 0.517 |
| 组合约束过滤率 | 30.4% | 33.1% |
| 组合约束正确通过率 | 65% | 67% |
| 条件方向预测准确率 | 71.7% | 72.3% |
| 独立方向预测准确率 | 47.1% | 47.7% |
| 条件 vs 独立提升 | +52.3% | +51.8% |

**对后续实验的指导：**
- E2 应优先纳入跨位置条件方向概率，而非组合统计约束
- 权重化框架（AUC 评估 + 差分条件分布）可作为后续实验的标准评估工具
- 组合约束可作为辅助手段，但不应期望大幅提升

### E2a：条件方向概率修正 [P0 - 已完成]

**状态**：已完成
**完成时间**：2026-02-08

**目的**：利用 E1.5 Step C 发现的跨位置方向关联（条件预测 72% vs 独立预测 47%），修正模型的方向概率输出，验证是否能提升方向预测准确率和缩号效率。

**方法**：
- 从训练期数据构建条件概率矩阵 P(d_{pos+1} | d_{pos})
- 用加权混合修正模型输出：P_final = (1-α)·P_model + α·P_conditional
- alpha 从 0.0 到 1.0 扫描，McNemar 检验显著性

**结论：条件方向概率修正对方向预测无显著提升，实验在 Step 1 终止。**

| 彩种 | 基线准确率 | 最优 alpha | 修正准确率 | 提升 | McNemar p |
|------|-----------|-----------|-----------|------|-----------|
| 大乐透 | 85.24% | 0.2 | 85.53% | +0.29% | 0.225（不显著） |
| 双色球 | 87.70% | 0.0 | 87.70% | 0% | 1.0（无变化） |

**关键分析**：
- 大乐透：alpha=0.2 时有微弱提升（+0.29%），但 McNemar 检验 p=0.225，远未达到 0.05 显著性水平
- 双色球：任何 alpha > 0 都导致准确率下降，条件概率修正完全无效
- alpha 越大准确率越低（alpha=1.0 时降至 70-74%），说明条件概率矩阵的信息质量远不如模型本身
- Step 0 稳定性验证：滚动窗口标准差多个位置对超过 0.05 阈值（未通过），但训练期 vs 测试期卡方检验全部通过

**原因分析**：
- E1.5 Step C 发现的 72% 条件预测准确率是相对于 47% 的独立（均匀）预测基线，而非相对于模型的 85%+ 预测
- 模型（XGBoost/MLP）已经隐式学习了位置间关联（通过局面特征中的多位置信息），条件概率矩阵提供的是冗余信息
- 条件概率矩阵是全局统计量，无法适应具体局面，而模型是局面条件化的预测

**对后续实验的指导**：
- 简单的后处理修正无法超越已训练好的模型，跨位置关联需要在模型训练阶段纳入（如多任务学习、联合预测）
- E2 应转向值域约束和组合约束等正交信息源，而非继续在方向预测上做后处理

结果文件：`src/research/results/experiment_strict/e2a_step{0,1}_*_{lottery_type}.json`
