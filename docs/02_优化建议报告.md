# 深度搜索 - 优化建议报告

> 文档生成时间: 2026-02-05
> 基于核心诉求: 分析问题核心 → 关联维度 → 清晰阐述 → 合并解答 → 适度扩散

---

## 一、核心诉求回顾

用户期望的深度搜索基础标准：

```
1. 分析出问题的核心是什么
2. 核心所关联的重要维度是什么
3. 每个维度要清晰阐述含义
4. 合并起来解答问题
5. 针对核心进行一定的扩散
```

---

## 二、当前实现与诉求的差距分析

| 诉求 | 当前实现 | 差距 |
|-----|---------|-----|
| **分析问题核心** | 识别 focus_entity（实体名称） | 只识别了"谁/什么"，未分析"用户真正想知道什么" |
| **关联重要维度** | 基于实体类型选择预设框架维度 | 维度是模板化的，未针对具体问题动态生成 |
| **清晰阐述维度含义** | 框架中有 description 字段 | 未在最终报告中展示维度含义，用户看不到 |
| **合并解答问题** | Writer 将所有资料合并写报告 | 合并是物理拼接，缺乏逻辑整合 |
| **针对核心扩散** | related_entities 做有限扩散 | 扩散是被动的（基于相关实体），缺乏主动发散 |

---

## 三、优化方案

### 3.1 新增「问题核心分析」阶段

**目标**: 在实体识别之前，先理解用户真正想知道什么

**新增 Step 0: 问题核心分析**

```python
class CoreAnalysis(BaseModel):
    """问题核心分析"""
    user_intent: str = Field(description="用户的真实意图，用一句话概括")
    core_question: str = Field(description="问题的核心是什么，提炼成一个精准的问题")
    question_type: str = Field(description="问题类型: 'what'(是什么), 'why'(为什么), 'how'(怎么做), 'who'(是谁), 'compare'(对比), 'evaluate'(评价)")
    depth_expectation: str = Field(description="用户期望的深度: 'overview'(概览), 'detailed'(详细), 'expert'(专家级)")
```

**示例**:

| 用户输入 | core_question | question_type |
|---------|---------------|---------------|
| "讲讲安禄山" | "安禄山是谁？他为什么重要？" | who + why |
| "安史之乱是怎么回事" | "安史之乱的起因、经过、影响是什么？" | what |
| "为什么唐朝会爆发安史之乱" | "安史之乱爆发的根本原因是什么？" | why |

### 3.2 重构「维度生成」逻辑

**目标**: 维度应该是针对具体问题动态生成的，而非模板套用

**当前流程**:
```
实体类型 → 选择预设框架 → 套用框架维度
```

**优化后流程**:
```
问题核心 → 分析需要哪些维度才能回答 → 动态生成维度 → 解释每个维度的含义
```

**新增 DimensionAnalysis 模型**:

```python
class Dimension(BaseModel):
    """单个分析维度"""
    name: str = Field(description="维度名称，如'崛起背景'、'性格动机'")
    why_important: str = Field(description="为什么这个维度对回答核心问题很重要")
    what_to_find: str = Field(description="在这个维度下需要找到什么信息")
    relation_to_core: str = Field(description="这个维度与核心问题的关系")

class DimensionPlan(BaseModel):
    """维度规划"""
    core_question: str = Field(description="核心问题")
    dimensions: List[Dimension] = Field(description="回答核心问题所需的维度列表")
    dimension_relations: str = Field(description="各维度之间的逻辑关系，如'时间顺序'、'因果链条'、'并列关系'")
```

**示例输出**:

```json

  "dimensions": [
    {
      "name": "身份背景",
```

### 3.3 优化「搜索结果整合」

**目标**: 搜索结果应该按维度组织，而非扁平化堆积

**当前数据结构**:
```python
research_data: List[str]  # 扁平化的字符串列表
```

**优化后数据结构**:
```python
class DimensionResult(BaseModel):
    dimension_name: str
    dimension_meaning: str  # 维度含义
    findings: List[str]     # 该维度下的发现
    confidence: str         # 信息充分度: 'high', 'medium', 'low'
    gaps: List[str]         # 信息缺口

research_data: List[DimensionResult]  # 按维度组织的结果
```

### 3.4 重构「报告生成」逻辑

**目标**: 报告应该清晰展示「核心问题 → 维度 → 发现 → 答案」的逻辑链

**新的报告结构**:

```markdown
# {核心问题的答案概括}

## 核心问题
{用户真正想知道什么}

## 维度分析

### 维度一：{维度名称}
**为什么重要**: {why_important}
**发现**: {findings}

### 维度二：{维度名称}
...

## 综合解答
{基于各维度发现，完整回答核心问题}

## 延伸思考
{针对核心的适度扩散}
```

---

### 3.5 新增「核心扩散」机制

**目标**: 在回答核心问题后，主动提供相关的延伸内容

**扩散策略**:

| 扩散类型 | 描述 | 示例 |
|---------|------|------|
| **时间扩散** | 前因后果 | "安禄山之前的藩镇问题" / "安史之乱后的唐朝" |
| **空间扩散** | 相关人物/事件 | "同时期的其他节度使" |
| **深度扩散** | 更深层的机制 | "唐朝藩镇制度的设计缺陷" |
| **对比扩散** | 类似案例 | "历史上其他以少胜多的叛乱" |

**实现方式**:

```python
class ExpansionPlan(BaseModel):
    """扩散计划"""
    expansion_type: str = Field(description="扩散类型")
    expansion_question: str = Field(description="扩散问题")
    relevance: str = Field(description="与核心问题的关联")
```
