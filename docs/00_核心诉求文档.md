# 深度搜索 - 核心诉求文档

> 文档创建时间: 2026-02-05
> 文档性质: 项目核心诉求定义，所有优化工作必须以此为基准

---

## 一、核心诉求定义

深度搜索的基础标准是：

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   1. 分析出问题的核心是什么                                       │
│                                                                 │
│   2. 核心所关联的重要维度是什么                                    │
│                                                                 │
│   3. 每个维度要清晰阐述含义                                       │
│                                                                 │
│   4. 合并起来解答问题                                            │
│                                                                 │
│   5. 针对核心进行一定的扩散                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 二、诉求详细解读

### 2.1 分析出问题的核心是什么

**定义**: 不是简单识别用户提到的实体，而是理解用户**真正想知道什么**。

**要求**:
- 将用户的模糊表述转化为精准的核心问题
- 识别问题的类型（是什么/为什么/怎么做/对比/评价）
- 判断用户期望的深度（概览/详细/专家级）

**示例**:

| 用户输入 | 核心问题 |
|---------|---------|
| "讲讲安禄山" | "安禄山是谁？他为什么在历史上重要？" |
| "安史之乱是怎么回事" | "安史之乱的起因、经过、影响分别是什么？" |
| "为什么唐朝会灭亡" | "唐朝灭亡的根本原因是什么？" |
| "React 和 Vue 哪个好" | "React 和 Vue 各自的优劣势是什么？适合什么场景？" |

**验收标准**:
- [ ] 系统能输出明确的 `core_question` 字段
- [ ] 核心问题比用户原始输入更精准、更具体
- [ ] 核心问题能指导后续的维度拆解

---

### 2.2 核心所关联的重要维度是什么

**定义**: 回答核心问题需要从哪些角度/方面进行分析。

**要求**:
- 维度应该是**针对具体问题动态生成**的，而非套用模板
- 维度之间应该有**逻辑关系**（时间顺序/因果链条/并列关系）
- 维度数量应该**适度**（通常 3-6 个），不宜过多

**示例**:

核心问题: "安禄山是谁？他为什么在历史上重要？"

| 维度 | 与核心的关系 |
|-----|-------------|
| 身份背景 | 回答"是谁"的基础 |
| 崛起之路 | 解释他如何获得发动叛乱的能力 |
| 叛乱动机 | 解释他为什么要反叛 |
| 历史影响 | 直接回答"为什么重要" |

**验收标准**:
- [ ] 每个维度都与核心问题有明确关联
- [ ] 维度是动态生成的，不同问题有不同维度
- [ ] 维度之间的逻辑关系被明确标注

---

### 2.3 每个维度要清晰阐述含义

**定义**: 用户应该能理解**为什么要从这个维度分析**，以及**这个维度要回答什么**。

**要求**:
- 每个维度必须有 `why_important` 字段：为什么这个维度重要
- 每个维度必须有 `what_to_find` 字段：这个维度要找什么信息
- 维度含义应该在最终报告中**对用户可见**

**示例**:

```
维度: 崛起之路

为什么重要: 一个边疆胡将如何获得皇帝的绝对信任，揭示了唐朝政治的深层问题。
          理解安禄山的崛起，才能理解他为什么有能力发动叛乱。

要找什么: 安禄山从底层士兵到三镇节度使的过程，他用了什么手段获得信任，
         唐玄宗为什么会信任他。
```

**验收标准**:
- [ ] 每个维度都有清晰的含义解释
- [ ] 含义解释能帮助用户理解分析逻辑
- [ ] 最终报告中展示维度含义

---

### 2.4 合并起来解答问题

**定义**: 各维度的发现应该被**逻辑整合**，形成对核心问题的完整回答。

**要求**:
- 不是简单的物理拼接，而是逻辑整合
- 最终报告应该有明确的「综合解答」部分
- 综合解答应该直接回应核心问题

**报告结构**:

```markdown
# {核心问题的答案概括}

## 核心问题
{精准的核心问题}

## 维度分析
### 维度一：{名称}
**为什么重要**: ...
**发现**: ...

### 维度二：{名称}
...

## 综合解答
{基于各维度发现，完整回答核心问题}
```

**验收标准**:
- [ ] 报告有明确的「综合解答」部分
- [ ] 综合解答直接回应核心问题
- [ ] 各维度发现被逻辑串联，而非简单罗列

---

### 2.5 针对核心进行一定的扩散

**定义**: 在回答核心问题后，主动提供相关的延伸内容，帮助用户拓展认知。

**要求**:
- 扩散内容与核心问题相关，但超出核心问题本身
- 扩散应该是**适度的**，不喧宾夺主
- 扩散类型可以包括：时间扩散、空间扩散、深度扩散、对比扩散

**扩散类型说明**:

| 类型 | 说明 | 示例 |
|-----|------|------|
| 时间扩散 | 前因后果 | "安史之乱后的唐朝走向" |
| 空间扩散 | 相关人物/事件 | "同时期的其他藩镇" |
| 深度扩散 | 更深层的机制 | "藩镇制度的设计缺陷" |
| 对比扩散 | 类似案例 | "历史上其他类似的叛乱" |

**验收标准**:
- [ ] 报告有「延伸思考」或类似部分
- [ ] 扩散内容与核心问题有明确关联
- [ ] 扩散内容能帮助用户拓展认知

---

## 三、诉求优先级

| 优先级 | 诉求 | 说明 |
|-------|------|------|
| **P0** | 分析问题核心 | 这是一切的基础，核心不清则后续全错 |
| **P0** | 关联重要维度 | 维度决定了分析的全面性和深度 |
| **P0** | 清晰阐述维度含义 | 让用户理解分析逻辑，而非被动接受结论 |
| **P1** | 合并解答问题 | 将分散的发现整合为完整答案 |
| **P2** | 针对核心扩散 | 锦上添花，帮助用户拓展认知 |

## 四、验收检查清单

每次优化后，使用以下检查清单验证是否满足核心诉求：

### 4.1 核心分析检查

- [ ] 系统是否输出了明确的核心问题？
- [ ] 核心问题是否比用户原始输入更精准？
- [ ] 核心问题是否能指导后续分析？

### 4.2 维度规划检查

- [ ] 维度是否针对具体问题动态生成？
- [ ] 每个维度是否与核心问题有明确关联？
- [ ] 维度之间是否有逻辑关系？
- [ ] 维度数量是否适度（3-6个）？

### 4.3 维度含义检查

- [ ] 每个维度是否有"为什么重要"的解释？
- [ ] 每个维度是否有"要找什么"的说明？
- [ ] 维度含义是否在最终报告中对用户可见？

### 4.4 综合解答检查

- [ ] 报告是否有明确的综合解答部分？
- [ ] 综合解答是否直接回应核心问题？
- [ ] 各维度发现是否被逻辑整合？

### 4.5 扩散检查

- [ ] 报告是否有延伸思考部分？
- [ ] 扩散内容是否与核心问题相关？
- [ ] 扩散是否适度，不喧宾夺主？

## 五、示例：理想的深度搜索输出

**用户输入**: "讲讲安禄山"

**理想输出**:

```markdown
# 安禄山：从胡人孤儿到颠覆盛唐的叛将

## 核心问题
安禄山是谁？他为什么在中国历史上如此重要？

## 维度分析

### 维度一：身份背景
**为什么重要**: 安禄山的胡人身份是理解他崛起和叛乱的关键。他的边缘人身份既是他被重用的原因，也是他最终反叛的伏笔。

**发现**: 安禄山（703-757），本姓康，是粟特人与突厥人的混血。出生于营州（今辽宁朝阳），幼年丧父，随母改嫁突厥人安延偃，遂改姓安。他精通六种语言，早年以互市牙郎为生...

### 维度二：崛起之路
**为什么重要**: 一个边疆胡将如何获得皇帝的绝对信任，揭示了唐朝政治的深层问题——皇帝对边将的过度依赖和对朝廷的不信任。

**发现**: 安禄山的崛起始于一次死里逃生。因盗羊被捕，临刑前高呼"我能杀两蕃"，打动了幽州节度使张守珪，不仅免死还被任命为"捉生将"...

### 维度三：叛乱动机
**为什么重要**: 理解安禄山为什么要反叛，才能理解安史之乱的本质——这不是一个人的野心，而是制度性危机的爆发。

**发现**: 关于安禄山的叛乱动机，史学界有多种解读...

### 维度四：历史影响
**为什么重要**: 安禄山的重要性不在于他本人，而在于他引发的安史之乱彻底改变了中国历史的走向。

**发现**: 安史之乱（755-763）是唐朝由盛转衰的转折点...

## 综合解答

安禄山是一个出身边疆的胡人将领，凭借军事才能和政治手腕获得唐玄宗的绝对信任，最终发动了改变中国历史走向的安史之乱。

他之所以重要，不仅因为他是叛乱的发动者，更因为他的崛起和叛乱暴露了唐朝政治制度的深层危机：皇帝对边将的过度依赖、藩镇权力的失控、中央与地方的矛盾。安史之乱后，唐朝虽然延续了150年，但再也没有恢复盛世的荣光，中国历史也从此进入了一个新的阶段。

## 延伸思考

- **安史之乱后的唐朝**: 为什么唐朝在安史之乱后还能延续150年？
- **藩镇制度的悖论**: 藩镇制度的设计初衷是什么？为何最终失控？
- **历史的对比**: 安禄山与其他历史上的叛将（如吴三桂）有何异同？
```
