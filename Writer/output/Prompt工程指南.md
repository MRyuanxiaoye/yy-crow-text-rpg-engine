# Prompt 工程指南

> 面向写作场景，结合黑乌鸦项目的实际经验。

---

## 一、Prompt 是什么

Prompt 就是你给模型的全部输入。模型根据这些输入，计算每个 token（字/词）的概率分布，然后逐个采样输出。

你给的输入越具体、越有方向性，概率分布就越集中在你想要的方向上。什么都不给，模型就输出统计中位数——也就是"AI味"。

Prompt 不是"命令"，是"起点"。你给的不是指令，是模型生成文本时的上下文环境。模型会基于这个环境，走概率最高的路径。你的工作是让"概率最高的路径"恰好是你想要的那条。

---

## 二、Prompt 能做什么

| 能力 | 说明 | 黑乌鸦项目中的对应 |
|------|------|---------------------|
| 设定角色 | 让模型以特定身份/视角输出 | `prompt.md` 中的角色身份定义 |
| 控制风格 | 拉偏概率分布，偏向特定作者的语感 | 语感片段（南派三叔/鲁迅） |
| 提供知识 | 给模型它训练数据中没有的信息 | 世界观设定、人物关系、前情 |
| 约束格式 | 控制输出的结构和形式 | 输出规范（文件命名、标注格式） |
| 定义流程 | 让模型按步骤执行复杂任务 | 写作流程（体验线→语感校准→分段写→审查） |
| 设定禁区 | 明确告诉模型什么不能做 | AI痕迹清单、"写之前不读corrections.md" |

---

## 三、Prompt 的组成结构

一个完整的 prompt 通常由以下几层构成，从外到内：

### 第1层：系统设定（System Prompt）

定义模型"是谁"。这是最底层的上下文，影响所有后续输出。

```
你是黑乌鸦，一个写手角色。
文风融合南派三叔和鲁迅。
南派三叔提供人物密度、世界质感、叙述者态度。
鲁迅提供简练、克制、重量感。
```

作用：把模型的概率分布从"通用AI"拉向"特定角色"。相当于给模型戴了一副有色眼镜，后续所有输出都会透过这副眼镜。

### 第2层：参考材料（Reference）

给模型具体的文本样本，让它"感受"你要的风格。

```
以下是南派三叔的语感片段，请在写作时保持类似的节奏和质感：

"老头把粥端过来的时候嘟囔了一句盐快没了，看守说跟方管事说，
老头说说了说明天让人去镇上买，然后就走了。"
```

作用：这是拉偏概率分布最有效的手段。模型在生成时会参考这些样本的句式、节奏、用词习惯，输出会向这些样本靠拢。

关键原则：
- 语感片段比规则描述有效10倍。"句子要短，节奏要硬"不如直接给一段鲁迅的原文
- 片段要精选，不是越多越好。5-10段高质量片段 > 50段泛泛的摘录
- 片段要按功能分类（对话、描写、收束、过渡），写不同类型的段落时加载对应类别

### 第3层：任务指令（Instruction）

告诉模型具体要做什么。

```
请写第五章。核心事件：河面一夜漂来十九具浮尸，营地全员捞尸。
陈渡在岸边被坠子触发，同时接收十几个人的临终记忆，晕倒。

体验线：混乱（营地炸锅）→ 沉重（捞尸细节）→ 崩溃（记忆洪流）→ 黑屏
```

作用：给模型一个明确的方向和终点。没有这个，模型会自己决定写什么，大概率写出一个平庸的、什么都沾一点的东西。

关键原则：
- 给结构，不给措辞。告诉模型"从A到B"，不要告诉它"第一句写什么"
- 体验线比大纲有效。"读者从混乱进入，经历沉重，在崩溃中结束"比"第一段写捞尸，第二段写记忆"好
- 核心事件只能有一个。多个事件会让模型在事件之间做无意义的过渡

### 第4层：约束条件（Constraints）

告诉模型什么不能做、什么要避免。

```
禁止：
- "不是A，是B"对比句式
- "他感到X"直接陈述情感
- 叙述者替读者判断
- 段落结尾刻意短句收束

要求：
- 每段500-800字，分段输出
- 冷热切换按需，不按固定模式
- 配角必须有具体的习惯或细节，不能只是功能性存在
```

作用：在概率分布上设置"路障"，阻止模型走向高概率但不想要的路径。

关键原则：
- 禁止项要具体，给例子。"不要AI味"没用，"不要用'不是A是B'句式"有用
- 禁止项不宜太多。超过10条模型就开始顾此失彼，注意力被分散
- 正面要求比负面禁止有效。"用身体反应代替情感陈述"比"不要直接写情感"好

### 第5层：上下文（Context）

当前任务的具体背景信息。

```
前情：
- 陈渡已在营地5天，坠子能触发临终记忆（第三章建立）
- 浮尸从零星变成批量（第四章铺垫）
- 方平川已经向上面报告了（第四章末尾）

当前场景的人物：
- 赵大勇：粗人，干活卖力，嘴里骂人但手上不停
- 孙嫂：嗓门大，干活利索，嘴硬心软
- 老头：沉默，脚受伤了还去烧水
- 小工：年轻，被吓到了，蹲在地上吐
```

作用：给模型足够的信息来保持连贯性。模型没有记忆，每次对话都是从零开始，你不给它就不知道。

## 四、Prompt 的设计原则

### 原则1：展示 > 描述

不要告诉模型"写得像南派三叔"，直接给它南派三叔的文本。

```
# 差
请用南派三叔的风格写作，特点是人物鲜活、对话自然、世界有质感。

# 好
请参考以下语感片段的节奏和质感来写作：
[直接贴5段南派三叔的原文]
```

原因：模型对"南派三叔风格"的理解是训练数据中所有提到南派三叔的文本的统计平均值，这个平均值很模糊。但你直接给它原文，它就能精确地锚定在那个节奏上。

### 原则2：具体 > 抽象

```
# 差
写一个紧张的捞尸场景。

# 好
河面上漂着十几具浮尸，营地所有人下水捞。天没亮，灯笼照在水面上晃。
写这个场景，从陈渡站在岸边的视角，他看到什么、听到什么、闻到什么。
```

原因：抽象指令给模型太多自由度，它会走概率最高的路径——也就是最平庸的那条。具体的起点把它推到一条特定的路径上。

### 原则3：结构化 > 自然语言

```
# 差
我想让你写一章，大概讲的是道士和僧人来找陈渡，方平川保护他，
最后三方达成协议。要有冷热切换，道士要有气势，僧人要安静，
方平川要有江湖气。字数3000左右。

# 好
## 核心事件
道佛两方来审视陈渡，方平川周旋保护。

## 体验线
被审视的压迫感 → 方平川的可靠感 → 僧人的异质感 → 暂时安全但危机升级

## 场景拆分
1. [热→冷] 醒来，方平川交代局势，宋先生探出气感
2. [热] 徐道士来探，被玉坠弹开，方平川挡人
3. [冷] 僧人来，不探不问，对视，留下一句话走了
4. [热→冷] 三方谈判结果，方平川说"有三个是本地的"

## 人物立场
- 徐道士：想带走陈渡，态度强势，被弹开后有忌惮
- 僧人：不带走，但要留下观察，态度平静但不可读
- 方平川：不想管但必须管，用江湖规矩挡两边
```

原因：结构化的 prompt 让模型清楚地知道每个部分的功能和要求，不会遗漏，也不会在不同要求之间混淆优先级。

### 原则4：分层加载，不要一次全给

这是黑乌鸦项目中最重要的经验。

```
# 差（一次性塞进去）
[角色设定] + [风格指南] + [AI痕迹清单] + [语感片段] + [前情] + [任务] + [约束]
全部放在一个 prompt 里

# 好（按阶段加载）
写之前：角色设定 + 语感片段 + 任务指令 + 前情
写的时候：只跟着语感走
写完之后：加载 AI痕迹清单 + 风格指南，做审查
```

原因：模型的注意力是有限的。信息越多，每条信息的权重越低。写作时同时加载"怎么写"和"不要怎么写"，模型会在两者之间摇摆，结果是两边都做不好。分阶段加载，每个阶段只给最相关的信息，效果最好。

### 原则5：给模型一个"起笔"

```
# 差
请写第五章。

# 好
请写第五章。开头第一句从这里开始：
"河边炸了锅。"
```

原因：第一句话决定了整个文本的概率路径。模型从第一个 token 开始，每个后续 token 都受前面所有 token 的影响。如果你让模型自己决定第一句，它大概率会选一个高概率的AI默认开头（"他是被X的"、"天亮的时候"）。你给它一个具体的、有力的起笔，后面的路径就被拉到了一个更好的方向上。

## 五、Prompt 的常见误区

### 误区1：规则越多越好

规则超过10条，模型就开始丢东西。它不是在"遵守规则"，是在概率空间里同时满足多个约束——约束越多，可行空间越小，模型越容易在约束之间打架，输出反而变差。

解决：把规则分成"写作时用的"和"审查时用的"，不要同时加载。

### 误区2：用自然语言写长段落的指令

```
# 差
我希望你在写作的时候注意一下，不要用太多的排比句，
尤其是那种"隔着A隔着B隔着C"的结构，这种在AI生成的文本中
非常常见，读起来很有AI味，所以请尽量避免......

# 好
禁止："隔着A，隔着B，隔着C"排比 → 只留最核心的一个
```

原因：模型处理长段落时会抓关键词，容易漏掉细节。短的、结构化的指令命中率更高。

### 误区3：期望 prompt 一劳永逸

没有一个 prompt 能让模型永远输出高质量内容。原因是语感衰减——生成越长，概率分布越往中位数滑。

解决：分段写 + 每段重新校准。这不是 prompt 的问题，是模型的底层机制决定的。

### 误区4：只用禁止，不给替代

```
# 差
不要用"他感到恐惧"这种直接陈述。

# 好
不要用"他感到恐惧"这种直接陈述。
替代方式：写身体反应。如"膝盖软了"、"手心全是汗"、"嗓子眼堵着"。
```

原因：你告诉模型不走A路，它不知道该走哪条，大概率走B路——另一条高概率的AI默认路径。你给它替代方案，它才知道往哪走。

## 六、黑乌鸦项目的 Prompt 架构

这是当前项目实际使用的分层结构，供参考：

```
┌─────────────────────────────────────────┐
│  第1层：角色设定（prompt.md）              │
│  "你是黑乌鸦，融合南派三叔和鲁迅"          │
├─────────────────────────────────────────┤
│  第2层：语感片段（写作时加载）              │
│  南派三叔语感片段 + 鲁迅语感片段            │
│  ※ 这是拉偏概率分布的核心武器              │
├─────────────────────────────────────────┤
│  第3层：任务指令（每章单独给）              │
│  核心事件 + 体验线 + 场景拆分 + 人物立场    │
├─────────────────────────────────────────┤
│  第4层：上下文（前情 + 世界观 + 人物状态）   │
├─────────────────────────────────────────┤
│  第5层：约束（写完后加载审查）              │
│  corrections.md + style_guide.md         │
│  ※ 写作时不加载，只在审查阶段使用          │
└─────────────────────────────────────────┘
```

这个架构的核心思路：写作阶段靠语感片段驱动，审查阶段靠规则约束。两个阶段分开，互不干扰。

## 七、实操建议

1. 先写一版，看看模型默认输出什么，再针对性地加约束。不要一上来就堆规则。
2. 语感片段是最值得投入时间的部分。花时间精选5-10段高质量片段，比写50条规则有效。
3. 每次只改一个变量。改了 prompt 之后输出变了，你才知道是哪个改动起了作用。
4. 保存有效的 prompt 模板。好的 prompt 是迭代出来的，不是一次写出来的。
5. 分段生成 + 中间审查，是对抗语感衰减的唯一可靠手段。这不是 prompt 能解决的问题，是使用流程的问题。
