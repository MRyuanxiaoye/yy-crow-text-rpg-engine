# 方案：文字RPG引擎 Phase 2 — 明末剧本 + 剧情系统

## 一、总体设计思路

- **骨架预写 + 细节LLM生成**：主线章节结构、关键事件节点、NPC人格卡预先写好（YAML），对话/叙事/分支后果由LLM实时生成
- **五种角色共享主线 + 各自专属支线**：大事件共用一条时间线骨架，每种角色有独有支线做差异感
- **条件触发推进**：数值阈值 + 关键决策完成 → 自动推进章节
- **玩家开局选择**：选角色 + 选主线长度（短/中/长）

## 六、NPC事件系统设计

每个NPC定义一组**生命周期事件**，与剧情节点绑定：

### 事件结构

```yaml
npc_name: 袁崇焕
events:
  - id: yuan_promoted
    name: 升任辽东督师
    chapter: 2
    trigger:
      type: chapter_enter  # 进入章节时自动触发
    effect:
      边防: +15
      narration: "袁崇焕走马上任，誓言五年平辽。"

  - id: yuan_impeached
    name: 被弹劾
    chapter: 3
    trigger:
      type: condition
      conditions:
        - 朝廷稳定 < 30
        - 回合 >= 8
    effect:
      朝廷稳定: -10
      npc_status: 被弹劾
      narration: "朝中弹章如雪片飞来..."

  - id: yuan_imprisoned
    name: 下狱
    chapter: 3
    trigger:
      type: decision  # 玩家做出"处置袁崇焕"决策后
      decision_id: D_yuan_fate
      choice: imprison
    effect:
      边防: -20
      军力: -10
      npc_status: 下狱

  - id: yuan_executed
    name: 被处死
    chapter: 3
    trigger:
      type: condition
      conditions:
        - npc_status == 下狱
        - 玩家未选择"赦免"
        - 回合 >= 10
    effect:
      边防: -15
      民心: -5
      npc_status: 死亡
      narration: "袁崇焕被凌迟处死于西市..."

  - id: yuan_pardoned
    name: 被赦免
    chapter: 3
    trigger:
      type: decision
      decision_id: D_yuan_fate
      choice: pardon
    effect:
      边防: +10
      朝廷稳定: -15
      npc_status: 复职
```

### 事件类型

| 类型 | 说明 |
|------|------|
| chapter_enter | 进入指定章节时自动触发 |
| condition | 满足数值/状态条件时触发 |
| decision | 玩家做出特定决策后触发 |
| npc_interaction | 与该NPC对话达到特定条件时触发 |

### NPC状态变化类型

死亡 / 下狱 / 降职 / 升迁 / 背叛 / 投降 / 流亡 / 结盟 / 反目 / 退隐 / 殉国

## 七、章节推进系统

### 推进逻辑

```
每次玩家操作后:
  1. 更新数值
  2. 检查当前章节的所有NPC事件 → 触发满足条件的
  3. 检查章节推进条件 → 满足则切换到下一章
  4. 章节切换时：
     a. 旁白发送过渡叙事
     b. 更新在场NPC（有人退场、有人登场）
     c. 触发新章节的 chapter_enter 事件
     d. 更新 game_date
```

### 短/中/长篇映射

| 长度 | 章节 | 每章回合数 | 总决策点 |
|------|------|-----------|---------|
| 短篇 | 3章（1→3→8） | 3-5 | 约9个 |
| 中篇 | 8章（完整） | 4-6 | 约24个 |
| 长篇 | 20+章（插入过渡章） | 5-8 | 约50+个 |

## 八、开局流程

```
玩家首次发消息 →
  1. 旁白卡片：游戏介绍 + "请选择你的身份"
  2. 展示五种角色（名称 + 一句话描述）
  3. 玩家选择角色
  4. 旁白卡片："请选择游戏长度"（短/中/长 + 说明）
  5. 玩家选择长度
  6. 加载对应剧本配置
  7. 发送开场叙事（根据角色视角定制）
  8. 进入第一章
```

## 十、多结局系统

### 结局判定时机
第8章（最终章）结束时，根据数值 + 关键NPC存活状态 + 玩家关键决策综合判定。

### 结局类型（每种角色4-6个结局）

**以崇祯皇帝为例：**

| 结局 | 条件 | 描述 |
|------|------|------|
| 中兴大明 | 国库≥60 且 军力≥60 且 民心≥50 | 力挽狂澜，开创新局 |
| 南渡偏安 | 军力<40 但 国库≥40 且 做出"南迁"决策 | 退守南方，延续国祚 |
| 煤山殉国 | 民心<20 且 军力<30 且 未南迁 | 历史重演 |
| 禅让求和 | 朝廷稳定<15 且 做出"议和"决策 | 屈辱求存 |
| 暴君覆灭 | 民心<10 且 朝廷稳定<10 | 众叛亲离 |
| 改革新政 | 民心≥70 且 朝廷稳定≥50 | 变法图强，走向不同历史 |

## 十一、技术实现要点

### 文件结构

```
scripts/
  mingmo/                    # 明末剧本
    manifest.yaml            # 剧本元信息（名称、角色列表、长度配置）
    chapters/
      ch01_ascend.yaml       # 各章节定义
      ch02_jisi.yaml
      ...
    fixed_npcs/
      袁崇焕.yaml
      李自成.yaml
      ...
    decisions/
      D_wei_party.yaml       # 决策点定义
      D_yuan_fate.yaml
      ...
    sidelines/
      emperor_harem.yaml     # 专属支线
      general_mutiny.yaml
      ...
    endings/
      emperor_endings.yaml   # 各角色结局定义
      ...
```

### 需要新增/改造的引擎模块

| 模块 | 改动 |
|------|------|
| engine/game_master.py | 增加章节推进检查、NPC事件检查 |
| engine/state.py | 增加角色专属数值、NPC状态追踪、章节进度 |
| engine/narrator.py | 增加章节过渡叙事、结局叙事 |
| engine/judge.py | 增加数值影响计算、NPC事件触发判定 |
| scripts/loader.py | 增加加载支线、结局、NPC事件的接口 |
| **新增** engine/progression.py | 章节推进引擎（条件检查+切换逻辑） |
| **新增** engine/npc_lifecycle.py | NPC生命周期管理（事件触发+状态变更） |
| **新增** engine/ending.py | 结局判定引擎 |

## 十二、实施顺序

1. **剧本数据编写**：manifest + 8章骨架 + 11个NPC人格卡+事件 + 决策点
2. **章节推进系统**：progression.py + state改造
3. **NPC生命周期**：npc_lifecycle.py + 事件触发
4. **开局选择流程**：角色选择 + 长度选择 + 视角定制开场
5. **专属支线**：支线加载 + 触发逻辑
6. **多结局系统**：ending.py + 结局叙事
